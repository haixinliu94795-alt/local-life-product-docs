# 行业模型到本平台模型映射

## 本章结论

本平台的底层领域模型采用标准清结算语言，不采用产品页面“待结算”宽口径，也不采用旧系统 `finance_detail` 反向建模。

| 行业概念 | 本平台领域语言 | P0 是否落地 | 说明 |
|---|---|---:|---|
| 业务事实 | `SourceEvent` | 是 | 核销/履约完成事件。 |
| 清分结果 | `ClearingResult` | 是 | 一次业务事实对应一次清分。 |
| 清分金额项 | `ClearingResultItem` | 是 | 支持多参与方、多金额项。 |
| 在途资金/头寸 | `SettlementPosition(IN_TRANSIT)` | 是 | 资金未结算前的标准头寸。 |
| 到期可结算 | `SettlementPosition(ELIGIBLE)` | 是 | 达到账期后可被结算。 |
| 结算锁定 | `LOCKED` | 是 | 被结算单锁定，防重复结算。 |
| 结算单 | `SettlementBill` | 是 | 平台对结算主体的一次结算单据。 |
| 账务入账单 | `AccountingPostingOrder` | 是 | 对账户账务平台的入账指令。 |
| 可提现余额 | 账户账务平台余额 | 外部 | 清结算平台不保存最终余额。 |
| 退款/冲正 | 逆向清分/调账 | 否 | 后续单独设计。 |
