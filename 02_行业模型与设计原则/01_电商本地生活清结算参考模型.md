# 电商 / 本地生活清结算参考模型

## 本章结论

成熟电商和本地生活平台的清结算不是“结算记录 CRUD”，而是把交易履约事实转化为可结算头寸，再生成结算单并驱动账务入账。平台化目标是支持多业务线、多参与方、多账期、多策略、可追溯和可补偿。

## 行业通用链路

```text
交易/履约事实
  -> 清分：计算多参与方金额归属
  -> 清算：判断账期成熟、聚合头寸、锁定可结算数据
  -> 结算：生成结算批次/结算单/明细
  -> 账务入账：写入商户账户、形成流水
  -> 查询和对账：支持商户、运营、财务追溯
```

## 对本平台的启示

| 行业能力 | 本平台对象 |
|---|---|
| 交易履约事实 | `SourceEvent` |
| 多参与方清分 | `ClearingResult`、`ClearingResultItem` |
| 在途/可结算头寸 | `SettlementPosition` |
| 账期策略 | `SettlementCyclePolicy` |
| 平台/商户/推广/成本策略 | `ClearingSettlementPolicy` |
| 结算单据 | `SettlementBatch`、`SettlementBill`、`SettlementBillItem` |
| 账务入账 | `AccountingPostingOrder` |
| 追溯和对账埋点 | 全链路编号和状态机 |
