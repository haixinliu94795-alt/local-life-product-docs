# 一期需求纳入范围

## 1. 范围原则

整体平台按成熟清结算平台设计；一期只落当前需求中最关键的普通商品清结算闭环。

```text
平台设计大面：业务接入、清分、清算、待结算、结算单、账务入账、出款衔接、对账差错、多业务接入。
一期实现小面：本地生活普通商品结算闭环。
```

## 2. 一期实现范围

| 能力 | 一期是否做 | 说明 |
|---|---:|---|
| 本地生活普通商品接入 | 是 | 从当前普通商品订单/核销数据接入。 |
| 清分结果 | 是 | 不能继续只依赖旧 finance_detail。 |
| 清分规则快照 | 是 | 支持比例、账期、优惠成本字段。 |
| 清算/待结算池 | 是 | 替代当前三张业务表 UNION 的待结算口径。 |
| 后台商家应付 | 是 | 满足当前页面需求。 |
| 结算批次/结算单/明细 | 是 | 清结算平台核心单据。 |
| 结算凭证 | 是 | 当前弹窗明确要求。 |
| 结算状态机 | 是 | 解决当前系统缺状态问题。 |
| 账务入账 | 是 | 复用账户账务平台。 |
| 商户端结算查询 | 是 | 待结算、已结算、详情。 |
| 提现/出款 | 否 | 外部平台边界；一期只衔接。 |
| 团餐/邀新/渠道 | 否 | 平台模型预留，P1 接入。 |
| 对账差错 | 最小 | 一期只做结算单 vs 账务流水校验，完整 P1。 |
