# 领域服务与领域事件

## 1. 领域服务

| 领域服务 | 职责 |
|---|---|
| SourceEventNormalizeService | 将本地生活、按摩、渠道等业务事件转换为标准清结算事件。 |
| ClearingSplitService | 基于规则快照计算清分结果和金额项。 |
| SettlementClearingService | 判断账期成熟，处理冻结/解冻，生成清算批次和待结算项。 |
| SettlementBillingService | 创建结算批次、结算单、结算明细和凭证，维护状态机。 |
| AccountingInstructionService | 生成账务入账指令、幂等键和重试策略。 |
| ReconciliationService | 多层对账和差异识别。 |

## 2. 应用服务

| 应用服务 | 用例 |
|---|---|
| ClearSettlementEventAppService | 业务事件接入、清分、进入待结算池。 |
| AdminPayableAppService | 后台商家应付查询、冻结/解冻、结算预览。 |
| SettlementOperationAppService | 运营确认结算、上传凭证、生成单据。 |
| AccountingOrchestrationAppService | 调账务入账、处理成功/失败/未知、重试。 |
| MerchantClearSettlementQueryAppService | 商户端待结算、已结算、结算详情、资金统计。 |
| RuleConfigAppService | 规则配置、发布、版本管理。 |
| ReconciliationAppService | 对账任务、差异处理。 |

## 3. 领域事件

| 领域事件 | 触发 | 消费 |
|---|---|---|
| SourceEventReceived | 标准事件接入成功 | 清分上下文 |
| ClearingResultCreated | 清分完成 | 清算上下文、对账上下文 |
| PendingSettlementItemCreated | 进入待结算池 | 查询投影、后台运营 |
| PendingSettlementItemMatured | 账期成熟 | 后台应付、自动批次任务 |
| SettlementBatchCreated | 生成批次 | 结算单上下文 |
| SettlementBillConfirmed | 确认结算 | 账务编排上下文 |
| AccountingPosted | 入账成功 | 查询投影、对账、通知 |
| AccountingFailed | 入账失败 | 后台运营、重试任务 |
| SettlementUnknown | 账务结果未知 | 人工核查、幂等查询任务 |
| ReconcileDiffFound | 发现差异 | 差错处理工作台 |
