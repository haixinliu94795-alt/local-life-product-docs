# 通用语言

| 术语 | 定义 |
|---|---|
| 清结算平台 | 统一承接交易后清分、清算、结算、账务入账编排和对账差错的平台。 |
| 清分 | 单笔交易维度的金额拆分，决定每个参与方、金额项和成本归属。 |
| 清算 | 周期维度的汇总、成熟判断、冻结/解冻、锁定、轧差和应收应付确认。 |
| 结算 | 生成结算单据并驱动账务入账，使商户或其他参与方形成可提现余额。 |
| 结算主体 | 最终需要形成结算单和入账的主体，如商户、门店、技师、渠道、服务商。 |
| 资金参与方 | 参与清分金额项的主体，可能是平台、商户、渠道、技师、服务商、推广者。 |
| 金额项 | 清分明细中的金额类型，如订单实付、商户应付、平台应收、优惠成本、佣金。 |
| 规则快照 | 清分时使用的规则版本快照，保证历史订单不受后续规则变更影响。 |
| 待结算项 | 已清分且满足或等待账期规则的最小结算候选项。 |
| 清算批次 | 对待结算项按商户、业务线、账期等维度汇总后的批次。 |
| 结算单 | 面向单一结算主体的应付/应收单据，是账务入账和商户展示的依据。 |
| 账务入账 | 调用账户账务平台生成余额变化和账户流水。 |
| 逆向清分 | 退款、取消、售后导致的负向金额拆分。 |
| 冲正 | 结算或入账后因错误或退款发生的反向调整。 |
| 对账 | 核对订单、清分、清算、结算、账务、出款之间的金额和状态一致性。 |
