# 架构表达标准：C4 + DDD + SDD

## C4 表达要求

| 层级 | 用途 | 本项目输出 |
|---|---|---|
| C1 System Context | 系统和外部系统边界 | `04_总体架构设计/01_C4_L1系统上下文.md` |
| C2 Container | 应用、模块、存储、外部端口 | `04_总体架构设计/02_C4_L2容器与模块.md` |
| C3 Component | 领域核心组件协作 | `04_总体架构设计/03_C4_L3核心组件.md` |
| Runtime | 关键场景如何运行 | `04_总体架构设计/04_运行时主链路.md` |

## DDD 表达要求

必须明确通用语言、限界上下文、聚合根、实体、值对象、领域服务、不变量、领域事件和上下文间依赖方向。

## SDD 表达要求

必须给出开发可以直接使用的 DDL、接口契约、DTO、状态机矩阵、幂等规则、错误码、测试验收和任务卡。
