# 高质量技术方案写作标准

## 本章结论

高质量技术方案不是“功能点 + 表 + 接口”的堆叠，而是从目标、约束、架构决策、领域边界、运行时流程、数据设计、异常补偿、测试验证形成闭环。

## 每个章节的固定表达结构

| 段落 | 必须回答 |
|---|---|
| 本章结论 | 本章最重要的设计判断是什么？ |
| 设计目标 | 解决什么问题，服务哪个质量属性？ |
| 范围与非范围 | 本章管什么，不管什么？ |
| 设计图 | 用图说明结构、流程或状态。 |
| 核心机制 | 实际怎么运行？ |
| 边界与失败路径 | 失败、重复、并发、异常怎么处理？ |
| 开发落点 | 代码放哪里，接口/表/类是什么？ |
| 验证方式 | 怎么测试，怎么验收？ |

## 禁止写法

1. 只列名词，不解释边界。
2. 只给 DDL，不解释字段、索引、查询场景。
3. 只给状态枚举，不给状态转移矩阵。
4. 只给接口路径，不给幂等、错误码、示例。
5. 只描述正常流程，不描述失败和补偿。
