# 清分规则模型

## 本章结论

清分规则不应写死商户/平台两方。P0 支持普通商品，模型上必须预留多参与方和多金额项。

| 字段 | 含义 |
|---|---|
| `participant_role` | MERCHANT / PLATFORM / CHANNEL / SERVICE_PROVIDER / PROMOTER / STORE / TECHNICIAN。 |
| `amount_code` | 金额项编码，如 ORDER_PAID、PLATFORM_COMMISSION、DISCOUNT_COST。 |
| `amount_direction` | RECEIVABLE / PAYABLE / COST / COMMISSION。 |
| `amount_cent` | 金额，单位分。 |
| `settlement_required` | 是否形成结算头寸。 |
