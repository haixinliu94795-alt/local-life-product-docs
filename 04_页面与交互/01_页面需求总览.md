# 页面需求总览

## 1. 页面分类

| 分类 | 页面 | 处理方式 |
| --- | --- | --- |
| 核心页面 | 商品管理_1、商品详情、商品已下架、商户已下架 | 作为本次需求来源。 |
| 关联页面 | 商家列表、商家详情页、订单、订单详情、交易快照、售后订单、售后订单详情 | 作为影响范围。 |
| 影响页面 | 财务首页、账户首页、待结算明细、结算详情页 | 只定义影响边界。 |
| 参考页面 | 折扣券、满减券等优惠券页面 | 仅参考上下架一致性，不纳入本期。 |

## 2. 页面设计原则

1. 用户端不应展示无法购买但仍诱导点击购买的入口。
2. 后台页面应明确展示对象状态和操作结果。
3. 有用户权益影响的操作必须有确认提示。
4. 原型未明确的页面行为不得在 PRD 中自行补全，应进入待确认清单。
