# 非 P0 容量与性能事项说明

## 1. 本章结论

本版本处于设计、开发、测试阶段，暂不进行容量规划和大数据量性能优化。所有性能相关内容只作为非 P0 说明，不进入开发准入范围。

## 2. P0 不覆盖

| 能力 | P0 处理 |
|---|---|
| 分库分表 | 不做 |
| 海量头寸成熟任务分片 | 不做 |
| 百万级导出 | 不做 |
| 异步导出中心 | 不做 |
| 冷热归档 | 不做 |
| 压测指标 | 不做 |
| 容量评估 | 不做 |
| 商户大户专属处理 | 不做 |

## 3. P0 保留的最小边界

虽然不做容量规划，但为了避免错误用法放大事故，P0 保留以下最小边界：

| 边界 | 数值 |
|---|---:|
| 账期成熟任务单轮处理数 | 200 |
| 确认结算单次头寸数 | 500 |
| 分页查询默认 pageSize | 20 |
| 分页查询最大 pageSize | 100 |

这些不是性能指标，而是开发期安全阈值。

## 4. 导出能力

`/eligible/export` 不属于清结算平台 P0。后台商家应付只提供分页查询。

如后续确实需要导出，应由运营 BFF、报表中心或异步导出能力承接，并单独设计导出限流、权限和审计。
