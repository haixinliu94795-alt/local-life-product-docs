# 统一响应与错误结构

## 1. 统一响应

```json
{
  "code": "SUCCESS",
  "message": "success",
  "traceId": "trace-xxx",
  "data": {}
}
```

## 2. P0 错误码

| 错误码 | 含义 | 处理建议 |
|---|---|---|
| `CCS_PARAM_INVALID` | 参数非法 | 调用方修正参数。 |
| `CCS_IDEMPOTENT_CONFLICT` | 同一幂等键请求指纹不一致 | 拒绝，人工排查。 |
| `CCS_BATCH_LIMIT_EXCEEDED` | 单次结算头寸数超过 500 | 缩小选择范围后分批提交。 |
| `CCS_POSITION_NOT_ELIGIBLE` | 头寸不可结算 | 重新查询可结算列表。 |
| `CCS_POSITION_MERCHANT_MISMATCH` | 批量头寸不属于同一商户 | 调整选择范围。 |
| `CCS_ACCOUNTING_UNKNOWN` | 账务结果未知 | 等待补偿任务或人工核查。 |
| `CCS_ACCOUNTING_FAILED` | 账务明确失败 | 按失败原因重试或人工处理。 |
| `CCS_OPERATION_NOT_ALLOWED` | 状态不允许当前操作 | 按状态机处理。 |

## 3. 幂等冲突响应示例

```json
{
  "code": "CCS_IDEMPOTENT_CONFLICT",
  "message": "same idempotentKey with different requestHash",
  "traceId": "trace-xxx",
  "data": {
    "idempotentKey": "SETTLE:xxx",
    "existingRequestHash": "abc",
    "currentRequestHash": "def"
  }
}
```
