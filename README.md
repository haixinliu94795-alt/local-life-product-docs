# 本地生活清结算平台 V005 SDD 开发准入版

本包是 V004 的领域标准收口版。V005 只保留清结算平台底层标准设计，不再让当前产品展示口径、旧系统字段、BFF 宽口径、退款未建能力或迁移时间点反向定义清结算平台。

## V005 关键收口

1. 删除 `cutover_time` 作为设计项。它只是上线迁移策略里的时间点，不是清结算领域字段，也不出现在 P0 核心方案中。
2. 删除旧 `finance_detail` 字段映射作为实现依据。P0 采用行业标准源事件、清分结果、结算头寸、结算单、账务入账模型。
3. 账务平台对接不再绑定 `AccountingFacade.recordProductSettlement`。V005 定义清结算平台需要的标准 `SettlementAccountingPort` 契约，账户账务平台应按该契约配合返回 `accounting_request_no` 和 `fund_account_flow_no`。
4. 退款、撤销、冲正不纳入 P0。P0 只做核销/履约完成后的正向清结算闭环，逆向清分后续单独建设。
5. 商户端 BFF 展示口径不纳入底层设计。清结算平台只提供标准读模型和状态，BFF 后续自行适配上层页面。
6. 将 `pending_settlement_item` 收口为更标准的 `settlement_position`，表达“结算头寸/待结算头寸”的行业语义。
