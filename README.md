# 本地生活清结算平台 SDD 开发设计 V002（行业标准版）

本版本用于修正 V001 中“从当前产品页面反推平台”的倾向，明确采用 **本地生活/电商成熟清结算平台** 的专业领域语言、资金状态模型和系统边界。

核心要求：

1. 平台底层模型不跟随当前产品文案走；当前页面和需求只作为一期接入场景。
2. 领域语言采用成熟行业口径：在途资金、清分、清算、结算、账务入账、可提现余额、提现/出款、对账差错。
3. 当前产品里的“待结算”属于前台展示口径，可由 BFF 聚合在途未到期、已到期可结算、结算处理中等多类底层状态。
4. 底层平台必须支持大规模运营：多业务线、多参与方、多账期、多规则快照、幂等、状态机、对账差错、审计追溯。
5. 一期实现只覆盖本地生活普通商品结算闭环，但领域模型要能扩展到团餐、邀新、渠道权益券、按摩平台等。
