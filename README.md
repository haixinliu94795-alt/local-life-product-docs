# 本地生活清结算平台 V009 SDD 正式开发方案 - 行业案例吸收与策略增强版

本版本在 V008 高质量重构版基础上，吸收个人博主结算案例文章中适合本地生活/电商清结算平台的通用能力：结算模式、结算周期、结算发起方、最小/最大结算金额、留存金额、规则绑定对象、客户账单与对账意识。

本版本不把传统行业案例直接搬进 P0，也不把发票、收款、付款、税务、出款并入清结算平台。它只把其中适合电商/本地生活平台的通用能力沉淀到 `ClearingSettlementPolicy` 策略上下文中。

## 本版结论

- 底层清结算平台仍采用标准主模型：`SourceEvent -> ClearingResult -> ClearingResultItem -> SettlementPosition -> SettlementBill -> AccountingPostingOrder`。
- 产品页面里的“结算配置”在底层命名为 **清分结算策略 ClearingSettlementPolicy**。
- 新增策略绑定模型：规则由业务线/类目/商户/活动等对象引用，不复制规则。
- 策略版本新增结算模式、发起方式、最小/最大金额、留存策略字段。
- P0 仍只做普通商品正向清结算：核销完成、清分、头寸成熟、后台手动结算、账户账务入账。
- P0 不做冻结/止付、退款/冲正、自动出款、发票/税务、完整对账中心、BFF 展示适配。
