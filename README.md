# 本地生活清结算平台 V010 SDD 正式开发方案 - P0 正确性闭环版

本包是在 V009「行业案例吸收与策略增强」基础上的 P0 正确性闭环修订版。

本版不做容量规划，不做大数据量性能方案，不做百万级导出，不做复杂调度框架；仅补齐会影响资金正确性、状态机闭环、幂等一致性和研发可落地性的关键设计。

## 本版核心修订

| 修订项 | 处理结果 |
|---|---|
| 账期成熟任务 | 新增 `SettlementPositionMaturityJob` 最小任务规格，保证 `IN_TRANSIT -> ELIGIBLE` 状态闭环。 |
| 批量结算边界 | `confirmMerchantSettlement` 单次最多 500 个 `positionNo`，超限拒绝，不做性能容量扩展。 |
| UNKNOWN 补偿 SLA | 定义自动核查间隔、逾期阈值、最大核查次数、人工介入条件，禁止 UNKNOWN 直接重复入账。 |
| 操作日志事务策略 | 核心资金事务提交后再写审计日志，日志失败不回滚资金主事务。 |
| 导出能力 | `/eligible/export` 不进入清结算平台 P0；P0 只做分页查询。 |
| request_hash | 标准化 SHA-256 + Canonical JSON + 字段集合，避免幂等冲突误判。 |
| 性能容量项 | 全部降级为非 P0 说明，不进入设计开发测试主线。 |

## 设计边界

P0 只覆盖本地生活/电商正向清结算闭环：来源事件、清分、结算头寸、账期成熟、结算确认、账务入账、失败补偿与开发验收。

P0 不包含退款、冻结/止付、出款、对账中心、百万级导出、分库分表、容量规划、复杂任务调度和 BFF 展示适配。
