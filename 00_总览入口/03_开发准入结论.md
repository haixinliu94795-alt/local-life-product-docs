# 开发准入结论

## 1. 结论

V005 满足一期普通商品正向清结算闭环的开发准入条件。

## 2. 准入范围

| 能力 | 是否准入 |
|---|---:|
| 标准履约事件接入 | 是 |
| 清分结果生成 | 是 |
| 结算头寸生命周期 | 是 |
| 商家应付可结算头寸查询 | 是 |
| 单条/批量统一确认结算 | 是 |
| 结算批次、结算单、结算明细 | 是 |
| 账务入账编排 | 是 |
| 账务失败/UNKNOWN 补偿 | 是 |
| 标准平台查询接口 | 是 |
| BFF 展示口径 | 否，后续适配 |
| 退款/冲正 | 否，后续逆向清分建设 |
| 提现/出款 | 否，外部平台负责 |
| 冻结/止付 | 否 |
| 旧系统字段映射 | 否 |

## 3. 开发前置依赖

1. 账户账务平台提供标准 `SettlementAccountingPort` 或等价适配器。
2. 业务方按清结算平台标准事件契约推送履约/核销完成事件。
3. 后台运营只使用清结算平台标准结算确认接口，不绕过状态机直接改表。
