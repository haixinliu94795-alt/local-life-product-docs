# V005 修订说明

## 1. 关于 cutover_time

`cutover_time` 在 V004 中用于表达旧链路与新链路的切换时间。V005 将其删除，原因如下：

- 它不是清结算平台领域概念；
- 它不应该进入 DDL、状态机或接口契约；
- 它属于上线迁移策略，由工程发布方案或配置平台负责；
- 本版设计不再以旧系统迁移为核心目标。

如未来上线需要灰度切换，应在部署方案中通过业务开关、路由策略或数据接入策略实现，不污染清结算平台领域模型。

## 2. 关于旧 finance_detail

V005 不再讨论旧 `finance_detail` 字段与新字段的逐项映射。清结算平台按照标准领域模型建设：

```text
Source Event -> Clearing Result -> Settlement Position -> Settlement Bill -> Accounting Posting Order
```

旧系统如何适配新标准事件，由接入适配器或一次性迁移工具解决，不属于平台底层模型。

## 3. 关于账务平台返回值

V005 不再受限于当前已有接口返回值。清结算平台对账户账务平台提出标准契约要求：

- 必须返回 `accounting_request_no`；
- 必须返回 `fund_account_flow_no`，或提供可查询到账户流水的等价流水引用；
- 必须支持幂等键和请求指纹校验；
- 必须支持按账务请求号查询最终结果。

## 4. 关于退款和提现

- 退款、撤销、冲正不纳入 P0；
- 提现/出款不是清结算平台主链路；
- 清结算平台 P0 只负责结算入账到账户账务平台，商户提现由出款/提现平台负责。

## 5. 关于商户端展示

V005 不设计商户端 BFF 宽口径待结算，也不判断页面是否展示 `ACCOUNTING_PROCESSING`。清结算平台只维护标准领域状态，上层展示由后续 BFF 层适配。
