# 模块：用户端展示控制

## 1. 模块目标

确保 C 端用户不会继续购买已下架商品或已下架店铺的新单，同时能够查看历史订单、交易快照和售后记录。

## 2. 展示控制口径

| 场景 | 产品口径 |
| --- | --- |
| 已下架商品出现在列表 | 不应展示在可购买列表。 |
| 用户直接访问已下架商品 | 展示已下架提示或不可购买态。 |
| 已下架店铺出现在列表 | 原则上不应作为可购买店铺展示，具体是否保留入口需确认。 |
| 用户直接访问已下架店铺 | 展示“谢谢光临，该店铺已下架”。 |
| 历史订单进入商品快照 | 允许访问交易快照，保留下单时信息。 |

## 3. 产品规则

1. 已下架对象不再进入新购买链路。
2. 用户端不能出现“可点击购买但最终报错”的体验，购买入口应前置收口。
3. 历史订单入口不应被下架状态阻断。
4. 交易快照应以订单发生时的商品信息为准，不因商品后续下架而变化。
5. 搜索、推荐、榜单、缓存等是否同步过滤已下架对象，需产品确认。

## 4. 页面来源

| 页面 | 关系 |
| --- | --- |
| 商品已下架 | 核心来源，已下架商品提示页。 |
| 商户已下架 | 核心来源，已下架店铺提示页。 |
| 商家列表 | 关联来源，正常商品/店铺曝光入口。 |
| 交易快照 | 关联来源，历史交易信息留痕。 |
