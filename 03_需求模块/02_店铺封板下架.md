# 模块：店铺封板/下架

## 1. 模块目标

当商户或店铺需要暂停展示、停止经营或被平台封板时，用户访问店铺应进入明确的已下架提示页，不能继续进入新购买链路。

## 2. 需求明细

| 编号 | 需求 | 优先级 | 说明 |
| --- | --- | --- | --- |
| REQ-STORE-001 | 店铺已下架用户端提示 | P0 | 用户访问已下架店铺时展示“谢谢光临，该店铺已下架”。 |
| REQ-STORE-002 | 店铺封板后台入口 | P0 | 后台操作入口、权限和恢复规则当前原型未完整体现，需确认。 |
| REQ-USER-001 | 用户端购买入口控制 | P0 | 已下架店铺不应继续展示可购买动作。 |

## 3. 产品规则

1. 店铺已下架后，用户端不再展示新购买入口。
2. 用户直接访问已下架店铺时，应展示明确提示，不应出现空白页或系统异常。
3. 店铺已下架不应删除用户历史订单。
4. 店铺已下架不应删除商家历史结算、提现、售后记录。
5. 店铺恢复营业是否需要重新审核，当前待确认。

## 4. 页面来源

| 页面 | 关系 |
| --- | --- |
| 商户已下架 | 核心来源，用户端店铺已下架提示。 |
| 商家列表 | 关联来源，正常店铺展示和购买入口。 |
| 商家详情页 | 关联来源，店铺详情和商品购买入口。 |

## 5. 待确认点

- 店铺封板入口在哪里：商户管理、店铺设置、风控后台还是运营后台？
- 店铺封板是否区分平台强制封板和商户主动暂停营业？
- 店铺恢复是否需要重新审核？
- 店铺封板后未核销订单是否还能核销？
