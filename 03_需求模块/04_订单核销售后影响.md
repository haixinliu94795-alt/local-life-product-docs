# 模块：订单、核销与售后影响

## 1. 模块目标

明确商品/店铺下架后，对已产生订单、核销和售后链路的产品影响，避免用户权益和交易留痕被破坏。

## 2. 产品口径

| 对象 | 处理口径 |
| --- | --- |
| 历史订单 | 不删除，继续可查。 |
| 订单详情 | 继续可查，包括订单金额、购买须知、核销信息等。 |
| 交易快照 | 保留下单时商品信息。 |
| 售后订单 | 不因商品/店铺下架而中断。 |
| 未消费订单 | 作为下架/删除限制条件，具体状态集合待确认。 |

## 3. 关键规则

1. 商品/店铺下架不应删除或隐藏用户历史订单。
2. 商品/店铺下架不应删除交易快照。
3. 售后申请、售后详情、退款状态、撤销退款等链路应继续可用。
4. 未消费订单是商品下架/删除的核心限制条件。
5. 待支付订单是否继续支付、待核销订单是否继续核销，需要进一步确认。

## 4. 待确认点

| 编号 | 问题 | 优先级 |
| --- | --- | --- |
| Q004 | “未消费订单”的准确范围是什么：待使用、退款中、部分核销、待付款是否包含？ | P0 |
| Q005 | 商品下架后，待付款订单是否允许继续支付？ | P0 |
| Q006 | 商品/店铺下架后，用户已购买未核销订单是否仍允许核销？ | P0 |
| Q007 | 商品/店铺下架后，售后申请、撤销、退款是否完整保留？ | P1 |

## 5. 页面来源

| 页面 | 关系 |
| --- | --- |
| 订单 | 历史订单列表。 |
| 订单详情 | 历史订单详情和售后入口。 |
| 交易快照 | 下单时信息留痕。 |
| 售后订单 | 售后状态展示。 |
| 售后订单详情 | 售后处理链路。 |
