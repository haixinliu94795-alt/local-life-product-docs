# 模块：财务与结算影响边界

## 1. 模块目标

本模块只定义产品影响边界，不展开结算平台、账务、提现或分账的技术设计。

## 2. 产品口径

| 场景 | 产品口径 |
| --- | --- |
| 已产生结算记录 | 不因商品/店铺下架而自动删除或改写。 |
| 已核销/已完成订单 | 原则上继续按原业务规则进入待结算或已结算链路。 |
| 退款中/争议中订单 | 是否进入待结算、冻结或冲减，需结合结算规则确认。 |
| 商户提现 | 是否因店铺封板冻结提现，当前原型未明确。 |
| 财务页面 | 原型中相关页面作为影响参考，不在本 PRD 中重写。 |

## 3. 明确不做

- 不定义结算平台化方案。
- 不定义账户账务模型。
- 不定义提现/出款流程。
- 不定义分账或对账规则。
- 不改写已有财务页面完整产品逻辑。

## 4. 需要评审的问题

| 编号 | 问题 | 优先级 |
| --- | --- | --- |
| Q008 | 商品/店铺下架后，已核销订单是否继续进入待结算/已结算？是否冻结提现？ | P1 |
| Q012 | 操作日志、操作人、操作时间、下架原因是否作为后台必填或审计字段？ | P1 |
