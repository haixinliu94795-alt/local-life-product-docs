# 模块：商品封板/下架

## 1. 模块目标

支持平台对商品执行下架、批量下架和上架操作，确保已下架商品不再对用户产生新交易，同时不破坏历史订单和交易记录。

## 2. 需求明细

| 编号 | 需求 | 优先级 | 说明 |
| --- | --- | --- | --- |
| REQ-GOODS-001 | 商品状态管理 | P0 | 商品列表支持全部、已上架、已下架、待审核、待上架等状态筛选和展示。 |
| REQ-GOODS-002 | 单个商品下架 | P0 | 商品列表或详情页可对单个商品执行下架。 |
| REQ-GOODS-003 | 批量商品下架 | P0 | 商品列表支持多选后批量下架。 |
| REQ-GOODS-004 | 商品上架与审核联动 | P0 | 未审核或审核未通过商品执行上架时，应先提交审核。 |
| REQ-GOODS-005 | 商品删除限制 | P1 | 商品存在未消费订单时，不允许删除；删除不删除历史订单。 |

## 3. 产品规则

1. 商品下架前必须校验商品是否存在未消费订单。
2. 单个商品存在未消费订单时，下架失败，并展示明确失败提示。
3. 批量下架时，存在未消费订单的商品自动过滤，仅处理允许下架的商品。
4. 已下架商品不应出现在用户可购买入口。
5. 已下架商品仍可在后台或历史记录中查看。
6. 商品上架如涉及审核，必须审核通过后恢复可售。
7. 商品删除不等于订单删除，有历史订单时必须保留交易信息。

## 4. 页面来源

| 页面 | 关系 |
| --- | --- |
| 商品管理_1 | 核心来源，包含状态筛选、批量上下架、删除限制和提示文案。 |
| 商品详情 | 核心来源，包含单个商品上下架和失败提示。 |
| 商品已下架 | 用户端结果页来源。 |

## 5. 待确认点

- 批量下架是否需要展示过滤数量和失败明细。
- 商户是否允许自主下架/上架商品。
- 商品上架审核通过后是否自动上架，还是需要再次确认。
