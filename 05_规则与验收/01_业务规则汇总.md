# 业务规则汇总

## 1. 商品规则

1. 商品状态至少包括：已上架、已下架、待审核、待上架、未通过、删除。
2. 商品已上架时可进入用户购买链路。
3. 商品已下架时不可进入用户购买链路。
4. 商品存在未消费订单时，单个下架失败。
5. 批量下架时，存在未消费订单的商品自动过滤。
6. 删除商品时，有未消费订单不可删除；订单信息不随商品删除而删除。
7. 商品上架如需审核，应审核通过后自动上架。

## 2. 店铺规则

1. 店铺已下架时，用户访问应看到已下架提示。
2. 店铺已下架时，不提供新购买入口。
3. 店铺已下架不删除历史订单、售后和结算记录。
4. 店铺恢复规则、是否重新审核、是否冻结结算/提现待确认。

## 3. 订单与售后规则

1. 商品/店铺下架不删除历史订单。
2. 交易快照保留下单时信息。
3. 售后链路不应因商品/店铺下架被中断。
4. 待付款订单、待核销订单、退款中订单的特殊口径待确认。

## 4. 财务规则

1. 商品/店铺下架不自动改写已有结算记录。
2. 已核销/已完成订单原则上继续按原业务规则进入结算。
3. 是否冻结待结算或提现需产品、财务和风控共同确认。
