# 状态口径

## 1. 商品状态

| 状态 | 产品口径 | 用户端 |
| --- | --- | --- |
| 已上架 | 商品已通过审核并可售。 | 可展示、可购买。 |
| 已下架 | 商品停止售卖。 | 不展示购买入口；直接访问显示不可购买或已下架。 |
| 待上架 | 商品未正式上架。 | 不可购买。 |
| 待审核 | 商品信息待审核。 | 不可购买。 |
| 未通过 | 商品审核未通过。 | 不可购买。 |
| 删除 | 后台删除或不可见状态，具体能力非本期主范围。 | 不可购买；历史订单保留。 |

## 2. 店铺状态

| 状态 | 产品口径 | 用户端 |
| --- | --- | --- |
| 正常营业 | 店铺可访问，可展示可售商品。 | 正常浏览和购买。 |
| 已下架/封板 | 店铺不可进入新交易链路。 | 展示已下架提示。 |
| 暂停营业 | 是否等同于已下架需确认。 | 待确认。 |

## 3. 未消费订单口径

原型明确使用“未消费订单”作为限制条件，但未定义状态集合。建议评审时确认是否包含：

- 待付款。
- 已付款待使用。
- 部分核销。
- 退款中。
- 争议中。
- 已预约未服务。
