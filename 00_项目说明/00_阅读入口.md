# 本地生活美食封板功能 - 产品项目文档 V001

## 1. 文档定位

本项目文档基于 Axure 原型交付包整理，定位为产品需求文档集，不包含技术方案、接口设计、数据库设计和测试用例。

本文档集的核心目标是：把原型中分散的“商品下架、商品已下架、商户已下架、订单/售后/结算影响”等内容，整理为可以评审、可以追溯、可以转交技术和测试的产品口径。

## 2. 推荐阅读顺序

| 顺序 | 文件 | 用途 |
| --- | --- | --- |
| 1 | `02_产品总览/00_产品需求总览.md` | 先看目标、范围、主流程和总需求。 |
| 2 | `03_需求模块/` | 按商品、店铺、用户端、订单售后、财务边界拆分阅读。 |
| 3 | `04_页面与交互/` | 对照原型页面确认页面行为和文案。 |
| 4 | `05_规则与验收/` | 看产品规则、异常口径和产品验收标准。 |
| 5 | `06_追溯与待确认/` | 看需求来源、待确认问题和原始证据。 |

## 3. 目录说明

| 目录 | 内容 |
| --- | --- |
| `01_原始输入/` | 用户上传的原始交付包及源文件 manifest。 |
| `02_产品总览/` | 项目级产品说明、范围、角色、核心流程和需求总表。 |
| `03_需求模块/` | 按能力域拆分的产品需求，不再集中在一个长文档中。 |
| `04_页面与交互/` | 页面级需求、交互行为和关键提示文案。 |
| `05_规则与验收/` | 产品规则、状态口径、验收标准。 |
| `06_追溯与待确认/` | 原型追溯矩阵、页面清单、待确认问题。 |
| `07_资产/` | 关键截图、页面总览和自动提取索引。 |

## 4. 当前版本结论

V001 版本可以作为产品初稿进入评审。当前最需要确认的是：店铺封板后台入口、恢复规则、未消费订单范围、商品/店铺下架后待支付/待核销/退款中订单的处理口径，以及结算/提现是否冻结。
