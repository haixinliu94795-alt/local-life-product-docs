# 文档边界

## 1. 本项目文档负责什么

- 明确“本地生活美食封板功能”的产品目标和范围。
- 将原型中出现的商品下架、店铺已下架、用户端已下架提示、历史订单保留、订单/售后/结算影响整理为产品口径。
- 明确不同角色的使用场景和页面行为。
- 给出可评审的业务规则、状态口径和产品验收标准。
- 标记待确认问题，避免在产品文档中自行补全未明确规则。
- 保留原型来源和关键截图，便于追溯。

## 2. 本项目文档不负责什么

| 不负责内容 | 原因 |
| --- | --- |
| 技术架构、接口设计、数据库设计 | 技术方案将在独立目录中承接。 |
| 测试用例、自动化测试、回归范围 | 测试文档将在独立目录中承接。 |
| 结算平台化设计 | 本功能只描述下架/封板对结算的产品影响边界。 |
| 商户入驻、商品新建、优惠券完整能力 | 原型中虽有关联页面，但非本次封板核心范围。 |
| 运营风控策略模型 | 本版本只定义前台/后台行为，不定义复杂风控策略。 |

## 3. 术语说明

| 术语 | 产品口径 |
| --- | --- |
| 封板 | 业务侧口头概念，本 PRD 中泛指停止店铺或商品继续对用户售卖/展示。 |
| 下架 | 原型明确出现的操作和状态，商品维度使用“下架/已下架”。 |
| 店铺已下架 | 原型中的用户端结果页，表示用户访问店铺时不再进入可购买链路。 |
| 未消费订单 | 原型明确提到的限制条件，具体状态集合待确认。 |
| 交易快照 | 下单时商品和交易信息留痕，不因商品下架或删除而消失。 |
