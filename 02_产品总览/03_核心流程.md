# 核心流程

## 1. 商品下架主流程

```text
进入商品管理/商品详情
  -> 选择单个商品下架
  -> 系统校验商品状态与未消费订单
  -> 无未消费订单
  -> 二次确认
  -> 下架成功
  -> 商品状态为已下架
  -> 用户端不可新购
```

## 2. 商品下架异常流程

```text
进入商品管理/商品详情
  -> 选择单个商品下架
  -> 系统发现存在未消费订单
  -> 下架失败
  -> 展示失败提示
  -> 商品状态不变
```

## 3. 批量下架流程

```text
商品列表选择多个商品
  -> 点击批量下架
  -> 系统判断每个商品是否存在未消费订单
  -> 过滤不可下架商品
  -> 可下架商品执行下架
  -> 返回处理结果
```

## 4. 商品上架流程

```text
选择已下架/待上架商品
  -> 点击上架
  -> 判断是否未审核或审核未通过
  -> 如需审核，则提交审核
  -> 审核通过后自动上架
  -> 恢复用户端购买入口
```

## 5. 店铺已下架用户端流程

```text
用户访问店铺
  -> 系统判断店铺状态为已下架
  -> 不展示新购买入口
  -> 展示已下架提示页
```

## 6. 历史交易保留流程

```text
商品/店铺下架
  -> 已产生订单不删除
  -> 订单详情继续可查
  -> 交易快照继续可查
  -> 售后记录继续可查
  -> 结算记录继续可查
```
