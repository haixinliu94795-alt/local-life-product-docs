# 背景、目标与范围

## 1. 背景

本地生活美食业务中，商品或店铺可能因为运营调整、合规风险、商家暂停经营、履约异常、售卖时间调整等原因停止对用户售卖。原型中出现了“商品下架”“商品已下架”“商户已下架”等页面和交互，本项目统一整理为封板/下架产品能力。

## 2. 产品目标

| 目标 | 说明 |
| --- | --- |
| 停止新增交易 | 已下架商品或已下架店铺不应继续进入新购买链路。 |
| 保留历史交易 | 已产生订单、交易快照、售后记录和结算记录不能因下架而丢失。 |
| 降低运营误操作 | 对存在未消费订单的商品进行限制，避免影响用户已购买权益。 |
| 统一用户体验 | 用户访问已下架商品/店铺时有明确提示，不出现空白或异常页面。 |
| 支持后续恢复 | 商品上架与审核流程联动，避免未经审核直接恢复售卖。 |

## 3. 本期范围

### 3.1 纳入范围

- 商品列表中的状态筛选和状态展示。
- 单个商品下架。
- 批量商品下架。
- 商品上架与审核联动。
- 商品删除时未消费订单限制。
- 用户端已下架商品/店铺提示。
- 历史订单、交易快照、售后记录保留。
- 对结算和提现影响的产品边界说明。

### 3.2 待确认后纳入

- 店铺封板/下架后台操作入口。
- 店铺恢复营业规则。
- 商户是否可自主下架/上架。
- 下架后待付款订单是否允许继续支付。
- 下架后待核销订单是否允许继续核销。
- 下架后是否冻结待结算或提现。

### 3.3 不纳入范围

- 商户入驻和资质审核完整流程。
- 商品新建、编辑、类目、品牌完整流程。
- 优惠券上下架完整能力。
- 财务结算、提现、分账系统重构。
- 技术方案、接口设计、数据库设计、测试用例。
