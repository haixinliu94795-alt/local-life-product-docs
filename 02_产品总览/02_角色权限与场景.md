# 角色、权限与使用场景

## 1. 角色列表

| 角色 | 说明 | 本功能关注点 |
| --- | --- | --- |
| 平台运营 | 管理本地生活商户、店铺、商品的后台人员。 | 商品/店铺下架、批量操作、异常原因、操作记录。 |
| 商户/供应商 | 商品或店铺的经营主体。 | 查看商品状态、维护商品信息、是否能自主操作待确认。 |
| C 端用户 | 浏览和购买本地生活商品的用户。 | 已下架商品/店铺不再可购买，历史订单仍可查看。 |
| 财务/运营 | 关注订单履约、结算、提现和售后留痕的人员。 | 下架动作不破坏历史资金链路。 |

## 2. 权限口径

| 操作 | 建议权限 | 当前状态 |
| --- | --- | --- |
| 单个商品下架 | 平台运营；商户是否可操作待确认 | 原型体现操作，但权限边界需确认。 |
| 批量商品下架 | 平台运营优先 | 原型体现批量确认文案。 |
| 商品上架 | 平台运营；商户是否可提交待确认 | 原型体现审核联动。 |
| 店铺封板/下架 | 平台运营/风控优先 | 后台入口需确认。 |
| 店铺恢复 | 平台运营/审核角色 | 恢复规则需确认。 |

## 3. 典型场景

### 场景 A：商品临时停售

运营在商品列表选择目标商品，下架后商品不再进入用户购买链路；历史订单继续可查。

### 场景 B：商品存在未消费订单

运营对商品发起下架，系统发现存在未消费订单，单个下架失败；若是批量下架，则过滤不可下架商品。

### 场景 C：店铺暂停营业

用户访问已下架店铺，看到“谢谢光临，该店铺已下架”，不再进入购买入口。后台封板入口和恢复规则需补充确认。

### 场景 D：下架后用户查看历史订单

用户仍可查看订单详情、交易快照和售后记录，避免交易凭证丢失。
