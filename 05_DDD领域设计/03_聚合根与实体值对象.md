# 聚合根与实体值对象

## 本章结论

聚合边界按资金事实生命周期划分。一个聚合只能维护自己的不变量，不能跨聚合直接修改字段。

| 聚合根 | 实体/值对象 | 关键不变量 |
|---|---|---|
| `SettlementPolicy` | `PolicyRule`、`ParticipantRule`、`RuleSnapshot` | 已发布策略不可被覆盖修改，只能新版本。 |
| `SourceEvent` | `SourceEventPayload` | 同业务键和事件类型只能有一个有效事件。 |
| `ClearingResult` | `ClearingResultItem` | 明细金额汇总必须与结果汇总一致。 |
| `SettlementPosition` | `SettlementCycle`、`PositionAmount` | 一个清分明细只能生成一个头寸。 |
| `SettlementBill` | `SettlementBillItem`、`SettlementBatch` | 结算单金额必须等于明细金额之和。 |
| `AccountingPostingOrder` | `AccountingPostingResult` | 一个结算单只能有一个有效入账单。 |

## V009 策略聚合增强

`ClearingSettlementPolicy` 聚合新增 `PolicyBinding` 作为独立实体。它不是策略聚合内部子对象，而是策略匹配上下文中的绑定关系实体，原因是绑定对象生命周期和策略版本生命周期不同。

### 新增不变量

1. `PolicyBinding` 只能绑定存在的 `policy_no`。
2. 同一绑定对象可存在多条绑定，但必须通过 `binding_priority` 和生效区间决出唯一命中策略。
3. `PolicyVersion` 发布后不可修改，只能生成新版本。
4. `ClearingResult`、`SettlementPosition`、`SettlementBill` 必须固化命中的 `policy_no` 和 `policy_version_no`。
5. P0 中 `settlement_mode` 只能为 `INTERNAL_ACCOUNT`，`initiation_mode` 只能为 `OPERATOR_MANUAL`。
