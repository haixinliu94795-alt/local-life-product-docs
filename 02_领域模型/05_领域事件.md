# 领域事件

## P0 入站事件

| 事件 | 来源 | 说明 |
|---|---|---|
| `WRITE_OFF_COMPLETED` | 本地生活核销/订单 | 核销完成，进入清分和待结算生成。 |
| `REFUND_SUCCESS` | 售后/退款 | 退款成功，触发结算前/中/后退款规则。 |
| `WRITE_OFF_CANCELED` | 核销/订单 | 取消核销，触发待结算项取消或人工处理。 |

## P0 内部领域事件

| 事件 | 触发点 | 用途 |
|---|---|---|
| `SourceEventAccepted` | 来源事件幂等落库成功 | 进入清分。 |
| `ClearingResultGenerated` | 清分结果生成 | 生成待结算项。 |
| `PendingItemEligible` | 账期成熟 | 后台商家应付可见。 |
| `SettlementBillConfirmed` | 平台确认结算 | 生成账务入账单。 |
| `AccountingPostingSucceeded` | 账务成功 | 推进结算单和待结算项为成功。 |
| `AccountingPostingFailed` | 账务失败 | 进入失败态，可重试。 |
| `AccountingPostingUnknown` | 账务结果未知 | 进入人工核查或查询补偿。 |

## Outbox 说明

P0 不建设对外事件 outbox。`ccs_source_event` 是入站 inbox。未来 P1 如需对商户 BFF、对账中心或运营通知发布事件，再新增 outbox 能力。
