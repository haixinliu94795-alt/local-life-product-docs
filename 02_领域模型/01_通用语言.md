# 通用语言

| 术语 | 定义 |
|---|---|
| 在途资金 | 支付/核销后尚未完成结算入账的资金事实。P0 表达为 `IN_TRANSIT` 待结算项。 |
| 清分 | 单笔订单维度，计算商户应结、平台应收、优惠成本等资金归属。 |
| 清算 | 清分之后、结算之前，对待结算项做账期成熟、汇总、锁定、金额校验和结算候选生成。 |
| 结算 | 生成结算批次、结算单、结算明细，并驱动账务入账。 |
| 可结算 | 达到账期且未结算、未取消、未锁定的数据，状态为 `ELIGIBLE`。 |
| 商家应付 | 后台运营视图，只展示可结算数据。 |
| 已结算 | 结算单已完成账户账务入账，状态为 `ACCOUNTED`。 |
| 可提现余额 | 账户账务平台维护的商户账户余额事实，不由清结算平台直接维护。 |
| 幂等键 | 表示一次业务提交意图的唯一键。 |
| 请求指纹 | `request_hash`，用于判断重复提交参数是否一致。 |

## 禁用口径

- P0 不使用 `currency / cur / currency_code`。
- P0 不使用 `tenant_id`。
- P0 不使用 `sourceSystem / sourceBizNo` 表达业务归属，统一使用 `caller_system / biz_domain / biz_no`。
