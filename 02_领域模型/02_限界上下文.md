# 限界上下文

## 1. 来源事件接入上下文

接收外部业务平台推送的标准履约完成事件，负责幂等、参数指纹校验和入站事件状态管理。

P0 接收事件：

```text
FULFILLMENT_CONFIRMED
```

不接收：退款、撤销、冲正、提现。

## 2. 清分上下文

根据来源事件生成清分结果和清分明细。清分上下文使用标准领域字段，不依赖旧系统表结构。

输出：

```text
ClearingResult
ClearingResultItem
```

## 3. 结算头寸上下文

将清分明细生成结算头寸，并维护在途、可结算、锁定、入账中、入账成功、入账失败、未知等状态。

输出：

```text
SettlementPosition
```

## 4. 结算单上下文

统一处理单条和批量结算确认，生成批次、结算单、结算明细。

核心命令：

```java
SettlementBillNo confirmMerchantSettlement(ConfirmMerchantSettlementCommand command);
```

## 5. 账务入账编排上下文

基于结算单生成账务入账单，调用账户账务平台标准端口，并处理成功、失败、未知状态。

## 6. 标准查询上下文

提供清结算平台标准查询接口，例如结算头寸查询、结算单查询、账务入账单查询。它不承诺页面展示口径。
