# 领域服务与不变量

## 1. 领域服务

| 服务 | 职责 |
|---|---|
| `SourceEventDomainService` | 来源事件幂等校验、请求指纹校验。 |
| `ClearingDomainService` | 生成清分结果和清分明细。 |
| `SettlementPositionDomainService` | 生成结算头寸、推进账期成熟、锁定/释放头寸。 |
| `SettlementBillingDomainService` | 统一确认结算，生成批次、结算单和明细。 |
| `AccountingPostingDomainService` | 发起账务入账、处理成功/失败/未知。 |

## 2. SettlementPosition 不变量

1. 同一个 `clearing_item_no` 只能生成一个 `position_no`。
2. `ACCOUNTED` 为终态，不允许回到任何非终态。
3. `LOCKED` 必须有关联 `locked_bill_no`。
4. `ACCOUNTING_PROCESSING` 必须有关联 `locked_bill_no` 和 `posting_no`。
5. `settlement_amount_cent >= 0`。
6. `ELIGIBLE` 才允许被确认结算锁定。
7. 取消结算单时，未进入账务处理的头寸必须释放回 `ELIGIBLE`。

## 3. SettlementBill 不变量

1. 一个 `bill_no` 只能对应一个 `batch_no`。
2. 一个结算单至少包含一条 `SettlementBillItem`。
3. `CREATED` / `CONFIRMED` 才允许取消。
4. `ACCOUNTING_PROCESSING` 不允许取消。
5. `ACCOUNTED` 为最终成功态。
6. `UNKNOWN` 只能通过账务查询补偿回正为 `ACCOUNTED` 或 `ACCOUNTING_FAILED`。
7. 重复确认结算时，同一 `idempotent_key` 且 `request_hash` 一致返回原 `bill_no`。
8. 同一 `idempotent_key` 但 `request_hash` 不一致必须拒绝。

## 4. AccountingPostingOrder 不变量

1. 一个 `bill_no` 只能有一个有效 `posting_no`。
2. `SUCCESS` 为终态。
3. `FAILED` 可重试。
4. `UNKNOWN` 必须先查询账户账务平台，不允许直接重复入账。
5. 每次调用账务平台必须复用同一个 `accounting_idempotent_key`。
