# 来源事件接入流程

## 本章结论

来源事件接入是全链路的入口，必须先完成幂等和 request_hash 校验，不能让重复或冲突事件进入清分。

## 流程

```text
接收 SourceEventCommand
  -> 校验 caller_system + biz_domain + biz_no + event_type
  -> 校验 idempotent_key
  -> request_hash 一致返回原事件
  -> request_hash 不一致拒绝
  -> 写 ccs_source_event(RECEIVED)
  -> 触发清分
```

## 状态

`RECEIVED -> PROCESSING -> PROCESSED / FAILED / IGNORED`
