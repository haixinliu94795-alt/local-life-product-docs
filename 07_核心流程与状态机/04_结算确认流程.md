# 结算确认流程

## 本章结论

单条结算和批量结算必须共用一个核心方法，区别只在 `positionNos` 数量。P0 批量策略为全成功/全失败。

## 统一方法

```java
SettlementBillNo confirmMerchantSettlement(ConfirmMerchantSettlementCommand command);
```

## 前置校验

1. `positionNos` 非空。
2. 全部为 `ELIGIBLE`。
3. 同一结算主体。
4. 同一业务场景。
5. 未被其他结算单锁定。
6. 金额汇总大于等于 0。
7. 幂等键和 request_hash 一致。

## 领域动作

```text
锁定头寸
  -> 创建 SettlementBatch
  -> 创建 SettlementBill
  -> 创建 SettlementBillItem
  -> 创建 AccountingPostingOrder
```
