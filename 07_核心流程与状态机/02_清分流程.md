# 清分流程

## 本章结论

清分负责从标准业务事实和策略快照生成多参与方清分结果，不负责账期成熟和结算单生成。

## 流程

```text
SourceEvent.PROCESSING
  -> 查询有效 PolicySnapshot
  -> 生成 ClearingResult
  -> 生成 ClearingResultItem
  -> 金额汇总校验
  -> 状态 CLEARED
  -> 生成 SettlementPosition
```

## 失败路径

1. 策略不存在：清分失败，事件进入 FAILED。
2. 金额不平：清分失败，保留错误码。
3. 重复清分：按 event_no 幂等返回原 clearing_no。
