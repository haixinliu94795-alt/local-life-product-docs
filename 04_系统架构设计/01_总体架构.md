# 总体架构

<img src="../assets/diagrams/01_capability_map.png" width="980" alt="清结算平台总体架构">

## 1. 架构分层

| 层 | 组件 | 说明 |
|---|---|---|
| 接口层 | Admin API、Merchant Query API、Event Consumer、Internal Facade | 协议转换、鉴权、参数校验、幂等入口。 |
| 应用层 | AppService、Command、Query、DTO | 编排业务用例，不沉淀核心规则。 |
| 领域层 | Aggregate、Entity、ValueObject、DomainService、DomainEvent | 清分、清算、结算、账务编排核心规则。 |
| 基础设施层 | Repository、MQ、File、Cache、AccountingClient、PayoutClient | 存储和外部系统适配。 |
| 防腐层 | LocalLifeAdapter、LegacyFinanceGateway、LegacySettlementGateway | 兼容旧订单表、旧结算表和旧接口。 |
| 查询投影层 | AdminPayableView、MerchantSettlementView、StatementView | 面向后台和商户端的读模型。 |
