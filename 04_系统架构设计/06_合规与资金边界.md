# 合规与资金边界

## 1. 清结算平台不做什么

| 不做事项 | 原因 |
|---|---|
| 不沉淀真实客户资金 | 真实资金应在支付机构、银行或合规账户体系中。 |
| 不直接银行卡出款 | 出款由提现/出款平台通过合规通道处理。 |
| 不保存账户余额最终事实 | 余额和流水由账户账务平台维护。 |
| 不绕过支付渠道做二清 | 清结算平台只做结算事实、账务指令和对账。 |

## 2. 应保存的事实

| 事实 | 所属平台 |
|---|---|
| 支付事实 | 支付平台 |
| 订单履约事实 | 业务/订单平台 |
| 金额归属事实 | 清结算平台 |
| 结算单据事实 | 清结算平台 |
| 余额流水事实 | 账户账务平台 |
| 提现出款事实 | 出款平台 |
| 对账差异事实 | 清结算平台/对应责任平台 |

## 3. 设计要求

- 所有结算事实必须可追溯到业务订单和清分明细。
- 所有账务入账必须可追溯到结算单和幂等键。
- 所有提现展示必须引用出款平台状态，不在清结算平台自造状态。
- 商户端金额口径必须明确：已结算不等于可提现余额，可提现余额以账户账务平台为准。
