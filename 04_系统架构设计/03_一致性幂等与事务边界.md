# 一致性、幂等与事务边界

## 1. 幂等键

| 层级 | 幂等键格式 | 说明 |
|---|---|---|
| 业务事件 | `EVENT:{scene}:{eventType}:{bizOrderNo}:{eventVersion}` | 防止重复接收业务事件。 |
| 清分 | `CLEARING:{scene}:{sourceOrderNo}:{ruleSnapshotNo}:{eventType}` | 防止重复清分。 |
| 清算项 | `PENDING:{clearingNo}:{participantId}:{amountType}` | 防止重复进入待结算。 |
| 结算批次 | `BATCH:{scene}:{merchantId}:{period}:{operatorOrJob}` | 防止重复生成批次。 |
| 结算单 | `BILL:{batchNo}:{merchantId}` | 防止重复生成结算单。 |
| 账务入账 | `ACCOUNTING:{billNo}` | 入账重试必须复用。 |

## 2. 事务边界

推荐分两段：

```text
事务 A：锁定待结算项 -> 创建结算批次/结算单/明细/凭证 -> 状态 CONFIRMED
事务 B：创建账务入账单 -> 调用账户账务平台 -> 根据结果更新状态
补偿任务：扫描 PROCESSING/UNKNOWN/FAILED 并按幂等键修复或重试
```

避免把外部账务调用放入一个大本地事务里长期持有锁。

## 3. 并发控制

| 场景 | 控制方式 |
|---|---|
| 两个运营同时结算同一待结算项 | `pending_item.status + version` 乐观锁，成功结算唯一索引。 |
| 重复点击确认结算 | `idempotent_key` 和结算单唯一索引。 |
| 账务接口超时后重试 | 按 `ACCOUNTING:{billNo}` 幂等查询或重试。 |
| 批量结算跨商户 | 后端重新校验，不信任前端选择。 |
| 金额被前端篡改 | 服务端按明细重新计算结算金额。 |
