# 读写分离与查询投影

## 1. 为什么要查询投影

后台商家应付和商户端财务页面需要高频分页、筛选和汇总，直接查写模型会导致复杂 JOIN 和性能风险。成熟平台通常将单据写模型和页面读模型分离。

## 2. 读模型

| 读模型 | 服务页面 | 来源 |
|---|---|---|
| admin_payable_view | 后台商家应付 | PendingItem + ClearingResult + SourceOrderSnapshot |
| admin_settlement_record_view | 后台结算记录 | SettlementBill + OperationLog + Voucher |
| merchant_finance_summary_view | 商户财务首页 | PendingItem + SettlementBill + Account + Payout |
| merchant_pending_detail_view | 商户待结算明细 | PendingItem |
| merchant_settlement_record_view | 商户结算记录 | ACCOUNTING_SUCCESS 的 SettlementBill |
| merchant_settlement_detail_view | 商户结算详情 | SettlementBill + SettlementBillItem |
| account_statement_enriched_view | 账户明细增强 | 账户账务流水 + 结算单关联信息 |

## 3. 更新方式

- 一期可以先同步更新投影，降低架构复杂度。
- P1 开始使用领域事件异步更新投影。
- 投影允许最终一致，但后台确认结算成功前必须以写模型状态为准。
