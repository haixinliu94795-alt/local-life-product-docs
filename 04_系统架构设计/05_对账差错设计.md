# 对账差错设计

## 1. 对账层级

| 对账层级 | 目的 | 一期 |
|---|---|---:|
| 订单/核销 vs 清分 | 防止支付或核销完成但未清分。 | 预留 |
| 清分 vs 清算 | 防止清分结果未进入待结算或重复进入。 | 预留 |
| 清算/待结算 vs 结算单 | 防止漏结、重复结、错结。 | 是，唯一索引 + 状态校验 |
| 结算单 vs 账务流水 | 防止结算成功但账务未入账，或账务入账但结算状态异常。 | 是，最小对账 |
| 账户流水 vs 出款 | 防止提现扣款与出款状态不一致。 | P1/P2 |
| 支付渠道账单 vs 支付单 | 防止支付掉单。 | 支付平台职责，清结算引用结果 |
| 退款单 vs 结算单 | 防止结算后退款资金处理错误。 | P2 |

## 2. 差异类型

- 漏清分；
- 重复清分；
- 清算金额不一致；
- 漏结算；
- 重复结算；
- 入账缺失；
- 重复入账；
- 状态不一致；
- 退款冲正缺失；
- 凭证缺失；
- 操作日志缺失。

## 3. 一期最小实现

一期至少保留这些对账键：

```text
source_event_no
source_order_no
clearing_no
pending_item_no
batch_no
bill_no
accounting_order_no
accounting_request_no
idempotent_key
```

并实现：

```text
SettlementBill(ACCOUNTING_SUCCESS) <-> fund_account_flow(category=SETTLEMENT)
```

的一致性校验。
