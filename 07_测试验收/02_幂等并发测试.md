# 幂等并发测试

| 场景 | 预期 |
|---|---|
| 同一核销事件重复接入 | 只生成一个 `event_no`、一个清分结果。 |
| 同一 `pending_no` 重复确认 | 返回原结算单或拒绝非法状态，不重复生成成功单。 |
| 同一 `idempotent_key` 同 `request_hash` | 返回原结果。 |
| 同一 `idempotent_key` 不同 `request_hash` | 返回 `CCS_IDEMPOTENT_CONFLICT`。 |
| 并发批量结算同一待结算项 | 只有一个事务锁定成功。 |
| 账务重试 | 复用同一 `accounting_idempotent_key`。 |
