# 新老双轨回归测试

| 场景 | 预期 |
|---|---|
| `write_off_time < cutover_time` | 仍走旧链路。 |
| `write_off_time >= cutover_time` | 走新清结算平台。 |
| 商户端待结算 | 合并旧 finance_detail + 新待结算宽口径。 |
| 商户端已结算 | 合并旧 local_settlement_record + 新 ACCOUNTED 结算单。 |
| 后台商家应付 | 只展示新平台 `ELIGIBLE`，旧链路收尾另行处理。 |
| 旧 Job | 切换后不得重复处理新平台订单。 |
