# 清结算平台 V005 开发准入总览

## 1. 版本定位

V005 是清结算平台 SDD 开发准入版，目标是指导 P0 普通商品正向清结算闭环开发。相比 V004，本版删除迁移口径、旧系统映射、退款最小规则和 BFF 展示适配，把底层模型收口到行业标准清结算领域语言。

## 2. 关键修订

| 项 | V005 结论 |
|---|---|
| `cutover_time` | 删除。它是上线迁移策略，不是领域字段。 |
| 旧 `finance_detail` 映射 | 删除。底层不按旧表字段建模。 |
| 账务接口 | 定义标准 `SettlementAccountingPort`，账户账务平台配合返回请求号和流水号。 |
| 退款 | 不纳入 P0。 |
| 提现 | 不纳入清结算主链路。 |
| 商户端 BFF | 不纳入底层设计。 |
| 核心模型 | `SourceEvent -> ClearingResult -> SettlementPosition -> SettlementBill -> AccountingPostingOrder`。 |

## 3. P0 主链路

```text
履约/核销完成事件
  -> 来源事件接入
  -> 清分结果
  -> 结算头寸
  -> 到期可结算
  -> 确认结算
  -> 结算批次/结算单/结算明细
  -> 账务入账单
  -> 账户账务平台入账
  -> 已入账成功
```

## 4. DDD 限界上下文

| 上下文 | 职责 |
|---|---|
| 来源事件接入 | 标准事件、幂等、请求指纹。 |
| 清分 | 订单级资金归属计算。 |
| 结算头寸 | 在途、可结算、锁定、入账中、失败、未知、成功。 |
| 结算单 | 批次、结算单、明细和状态机。 |
| 账务入账 | 调用账户账务平台并处理回写。 |
| 标准查询 | 提供底层标准查询，不承诺页面展示。 |

## 5. 状态机

### 结算头寸

```text
IN_TRANSIT -> ELIGIBLE -> LOCKED -> ACCOUNTING_PROCESSING -> ACCOUNTED
                                      |-> ACCOUNTING_FAILED -> ACCOUNTING_PROCESSING
                                      |-> UNKNOWN -> ACCOUNTED/ACCOUNTING_FAILED
```

### 结算单

```text
CREATED -> CONFIRMED -> ACCOUNTING_PROCESSING -> ACCOUNTED
                                      |-> ACCOUNTING_FAILED -> ACCOUNTING_PROCESSING
                                      |-> UNKNOWN -> ACCOUNTED/ACCOUNTING_FAILED
CREATED/CONFIRMED -> CANCELED
```

### 账务入账单

```text
INIT -> REQUESTING -> SUCCESS
                  |-> FAILED -> REQUESTING
                  |-> UNKNOWN -> SUCCESS/FAILED
```

## 6. 数据模型

P0 核心表：

```text
ccs_source_event
ccs_clearing_result
ccs_clearing_result_item
ccs_settlement_position
ccs_settlement_batch
ccs_settlement_bill
ccs_settlement_bill_item
ccs_accounting_posting_order
ccs_settlement_operation_log
```

DDL 规范：金额全部 `BIGINT` 分；无 `currency`；无 `tenant_id`；无退款字段；无旧系统映射字段。

## 7. 账务平台要求

账户账务平台需要提供标准端口：

```java
SettlementAccountingPostResult postSettlement(SettlementAccountingPostCommand command);
SettlementAccountingPostResult querySettlementPosting(String accountingRequestNo);
```

成功时必须返回：

```text
accounting_request_no
fund_account_flow_no
```

如果结果未知，清结算平台进入 UNKNOWN，先查询账务平台，不允许直接重复入账。

## 8. 开发准入结论

V005 满足 P0 普通商品正向清结算开发准入。开发时不得加入旧系统映射、BFF 页面口径、退款、提现、冻结、止付和出款 TRANSFER 主链路。
