# 结算头寸状态机

## 1. 状态定义

| 状态 | 说明 | 是否可结算 | 是否终态 |
|---|---|---:|---:|
| `IN_TRANSIT` | 在途，履约已完成但未到可结算时间。 | 否 | 否 |
| `ELIGIBLE` | 已到账期，可被结算。 | 是 | 否 |
| `LOCKED` | 已被结算单锁定，防重复结算。 | 否 | 否 |
| `ACCOUNTING_PROCESSING` | 对应结算单正在账务入账。 | 否 | 否 |
| `ACCOUNTING_FAILED` | 账务入账失败，可随结算单重试。 | 否 | 否 |
| `UNKNOWN` | 账务结果未知，需先查询账务平台。 | 否 | 否 |
| `ACCOUNTED` | 已入账成功。 | 否 | 是 |

## 2. 状态流转矩阵

| 当前状态 | 触发事件 | 前置条件 | 领域动作 | 目标状态 | 失败目标状态 | 是否幂等 | 是否可重试 | 测试用例 |
|---|---|---|---|---|---|---|---|---|
| - | createPosition | 清分明细生成 | 创建结算头寸 | `IN_TRANSIT` | - | 是 | 是 | TC-POS-001 |
| `IN_TRANSIT` | mature | `eligible_time <= now` | 标记可结算 | `ELIGIBLE` | `IN_TRANSIT` | 是 | 是 | TC-POS-002 |
| `ELIGIBLE` | lockForBill | 结算确认命令校验通过 | 写入 `locked_bill_no` | `LOCKED` | `ELIGIBLE` | 是 | 是 | TC-POS-003 |
| `LOCKED` | startAccounting | 结算单进入入账 | 写入 `posting_no` | `ACCOUNTING_PROCESSING` | `LOCKED` | 是 | 是 | TC-POS-004 |
| `ACCOUNTING_PROCESSING` | accountingSuccess | 账务成功 | 标记入账成功时间 | `ACCOUNTED` | `UNKNOWN` | 是 | 是 | TC-POS-005 |
| `ACCOUNTING_PROCESSING` | accountingFailed | 账务明确失败 | 记录失败原因 | `ACCOUNTING_FAILED` | `UNKNOWN` | 是 | 是 | TC-POS-006 |
| `ACCOUNTING_PROCESSING` | accountingUnknown | 调用超时或结果不明 | 记录待查询 | `UNKNOWN` | `UNKNOWN` | 是 | 是 | TC-POS-007 |
| `ACCOUNTING_FAILED` | retryAccounting | 结算单重试 | 跟随结算单重新入账 | `ACCOUNTING_PROCESSING` | `ACCOUNTING_FAILED` | 是 | 是 | TC-POS-008 |
| `UNKNOWN` | queryAccountingSuccess | 查询到账务成功 | 回正成功 | `ACCOUNTED` | `UNKNOWN` | 是 | 是 | TC-POS-009 |
| `UNKNOWN` | queryAccountingFailed | 查询到账务失败 | 回正失败 | `ACCOUNTING_FAILED` | `UNKNOWN` | 是 | 是 | TC-POS-010 |
| `LOCKED` | cancelBillBeforeAccounting | 结算单未入账前取消 | 释放锁定 | `ELIGIBLE` | `LOCKED` | 是 | 否 | TC-POS-011 |

## 3. 禁止流转

- `ACCOUNTED` 不能回到任何非终态。
- `ACCOUNTING_PROCESSING` 不能被运营直接取消。
- `UNKNOWN` 不能直接重复入账，必须先查账务平台。
- `ELIGIBLE` 之外的头寸不能被新结算单锁定。
