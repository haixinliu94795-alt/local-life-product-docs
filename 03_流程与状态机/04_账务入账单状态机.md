# 账务入账单状态机

## 1. 状态定义

| 状态 | 说明 | 是否可重试 | 是否终态 |
|---|---|---:|---:|
| `INIT` | 已创建，尚未请求账务。 | 是 | 否 |
| `REQUESTING` | 正在请求账务平台。 | 否 | 否 |
| `SUCCESS` | 账务入账成功。 | 否 | 是 |
| `FAILED` | 账务明确失败。 | 是 | 否 |
| `UNKNOWN` | 账务结果未知。 | 需先查询 | 否 |

## 2. 状态流转矩阵

| 当前状态 | 触发事件 | 前置条件 | 领域动作 | 目标状态 | 失败目标状态 | 是否幂等 | 是否可重试 | 测试用例 |
|---|---|---|---|---|---|---|---|---|
| - | createPostingOrder | 结算单已确认 | 创建入账单 | `INIT` | - | 是 | 是 | TC-POST-001 |
| `INIT` | requestAccounting | 入账参数完整 | 调用账务平台 | `REQUESTING` | `FAILED` | 是 | 是 | TC-POST-002 |
| `REQUESTING` | accountingSuccess | 返回账务请求号和流水号 | 保存回写编号 | `SUCCESS` | `UNKNOWN` | 是 | 是 | TC-POST-003 |
| `REQUESTING` | accountingFailed | 明确失败 | 保存失败原因 | `FAILED` | `UNKNOWN` | 是 | 是 | TC-POST-004 |
| `REQUESTING` | accountingTimeout | 超时/网络异常 | 标记未知 | `UNKNOWN` | `UNKNOWN` | 是 | 是 | TC-POST-005 |
| `FAILED` | retry | 失败可重试 | 复用账务幂等键 | `REQUESTING` | `FAILED` | 是 | 是 | TC-POST-006 |
| `UNKNOWN` | querySuccess | 查询到账务成功 | 回正成功 | `SUCCESS` | `UNKNOWN` | 是 | 是 | TC-POST-007 |
| `UNKNOWN` | queryFailed | 查询到账务失败 | 回正失败 | `FAILED` | `UNKNOWN` | 是 | 是 | TC-POST-008 |

## 3. 禁止流转

- `SUCCESS` 不允许重试。
- `UNKNOWN` 不允许直接再次入账，必须先查询账务平台。
- 重试必须复用同一个 `accounting_idempotent_key`。
