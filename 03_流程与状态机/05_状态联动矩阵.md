# 状态联动矩阵

| 业务动作 | 待结算项 | 结算单 | 入账单 |
|---|---|---|---|
| 核销完成生成待结算 | `IN_TRANSIT` | - | - |
| 账期成熟 | `ELIGIBLE` | - | - |
| 确认结算并锁定 | `LOCKED` | `CREATED -> CONFIRMED` | `INIT` |
| 开始入账 | `ACCOUNTING_PROCESSING` | `ACCOUNTING_PROCESSING` | `REQUESTING` |
| 账务成功 | `ACCOUNTED` | `ACCOUNTED` | `SUCCESS` |
| 账务失败 | `ACCOUNTING_FAILED` | `ACCOUNTING_FAILED` | `FAILED` |
| 账务未知 | `ACCOUNTING_PROCESSING` | `UNKNOWN` | `UNKNOWN` |
| 失败重试 | `ACCOUNTING_PROCESSING` | `ACCOUNTING_PROCESSING` | `REQUESTING` |
| 未入账取消 | `ELIGIBLE` 或 `CANCELED` | `CANCELED` | - |

## 联动要求

- 结算单成功后，所有明细对应待结算项必须成功。
- 入账失败时，结算单和待结算项必须同时进入失败态。
- UNKNOWN 只表示账务结果未知，不表示业务失败。
