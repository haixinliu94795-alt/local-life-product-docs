# P0 范围与非范围

## 1. P0 范围

| 能力 | 说明 |
|---|---|
| 履约/核销完成事件接入 | 接收标准履约完成事件，不绑定旧系统字段。 |
| 订单级清分结果 | 根据标准输入生成 `ClearingResult` 和 `ClearingResultItem`。 |
| 结算头寸生成 | 将清分明细生成 `SettlementPosition`。 |
| 账期成熟判断 | 通过 `eligible_time` 判断头寸是否可结算。 |
| 商家应付 | 后台运营查询 `ELIGIBLE` 状态的结算头寸。 |
| 统一结算确认 | 单条和批量走同一个 `confirmMerchantSettlement`。 |
| 结算单状态机 | 生成批次、结算单、结算明细并推进状态。 |
| 账务入账编排 | 通过标准账务端口入账到商户账户。 |
| 账务异常补偿 | 支持 FAILED 重试，UNKNOWN 先查账务再处置。 |
| 标准查询接口 | 提供底层标准领域查询能力，不承诺具体页面口径。 |

## 2. P0 非范围

| 非范围 | 说明 |
|---|---|
| 退款/撤销/冲正 | 后续逆向清分能力。 |
| 提现/出款 | 出款/提现平台负责。 |
| 冻结/止付 | 风控/争议能力，P0 不做。 |
| BFF 展示适配 | 上层展示不进入底层设计。 |
| 旧系统字段映射 | 接入适配器问题，不进入清结算平台模型。 |
| 历史全量迁移 | 不在本版设计范围。 |
| 自动结算任务 | P1，可基于同一结算确认服务扩展。 |
| 对账差错工作台 | P1/P2。P0 只保留账务回写与状态补偿。 |
