# 外部平台边界

| 外部系统 | 清结算平台调用方式 | P0 说明 |
|---|---|---|
| 本地生活订单/核销 | 接收核销完成、退款成功、取消核销事件 | P0 的业务事件来源。 |
| 账户账务平台 | 调用 `AccountingFacade.recordProductSettlement` | 结算成功后入账到商户账户。 |
| 出款/提现平台 | 仅由 BFF 查询提现中/已提现金额 | 清结算 P0 不发起出款。 |
| 商户端 BFF | 查询清结算读模型 + 账户账务 + 出款 | 适配当前产品展示口径。 |
| 后台运营 | 查询商家应付、发起结算、重试、人工核查 | 只允许对 `ELIGIBLE` 发起结算。 |

## TRANSFER 边界

普通商品 P0 不接入出款核心 TRANSFER。未来如需要平台应收实收或商户向平台支付平台佣金，应由清结算平台生成平台应收指令，再通过独立适配器对接出款核心；该能力不得污染 P0 商户应结入账链路。
