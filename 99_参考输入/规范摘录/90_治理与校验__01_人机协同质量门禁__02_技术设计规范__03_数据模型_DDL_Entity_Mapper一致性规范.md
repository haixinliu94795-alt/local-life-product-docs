# 数据模型、DDL、Entity、Mapper 一致性规范

## 1. 一致性目标

数据表、代码实体、Mapper 查询、DTO 转换必须一致，防止出现运行时才发现字段不存在、字段未写入、旧字段残留的问题。

## 2. 必须对齐的对象

| 层 | 检查内容 |
|---|---|
| DDL | 字段、类型、默认值、非空、唯一约束、索引 |
| Entity | 字段名、映射列、是否存在表字段、是否忽略非表字段 |
| Mapper XML | 查询列、插入列、更新列、where 条件 |
| QueryWrapper / LambdaQuery | 是否引用废弃字段或不存在字段 |
| DTO / VO | 是否暴露不该暴露的内部字段 |
| Converter | 字段来源是否正确，是否混用语义 |
| Migration | 表结构升级是否可重复、可回滚、可验证 |

## 3. 自动化检查要求

CI 至少应能输出：

```text
表名
Entity 类
DDL 字段集合
Entity 映射字段集合
Mapper 引用字段集合
多余字段
缺失字段
废弃字段命中位置
是否阻断发布
```

核心表出现以下情况必须失败：

- Entity 映射字段在 DDL 中不存在；
- Mapper 查询字段在 DDL 中不存在；
- DDL 非空字段没有写入来源；
- 已废弃字段仍被业务逻辑查询；
- 已废弃字段作为主关联或幂等字段使用。

## 4. 非表字段处理

如果 Entity 中存在非表字段，必须显式标识为不落表字段，并说明用途。

禁止让 ORM 自动把临时字段、解析字段、幂等中间字段当作数据库列查询。

## 5. DDL 变更准出

涉及 DDL 的变更发布前必须完成：

1. 方案字段契约更新；
2. DDL 脚本更新；
3. Entity / Mapper / Repository 更新；
4. 单测和 Repository 测试更新；
5. 线上环境字段核验脚本；
6. 回滚或兼容策略。
