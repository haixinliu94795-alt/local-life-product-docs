# Codex 全局硬约束

## 1. MUST

| 约束 | 决策ID |
|---|---|
| 写入类跨平台请求必须区分 `callerSystem`、`bizDomain`、`bizNo`，不得用技术调用方替代业务归属域。 | `DEC-BIZ-001` ~ `DEC-BIZ-005` |
| 资金出款账务处理按 `applyAmount` 调用账户账务冻结、扣减冻结、解冻。 | `DEC-AMT-001`、`DEC-AMT-004` |
| 出款核心负责提现通道金额策略，支付通道网关只接收已换算完成的通道金额。 | `DEC-AMT-005`、`DEC-AMT-006` |
| 出款结果必须由出款核心统一收敛状态并发布事件。 | `DEC-PO-006` |
| 商家账户明细展示已结算、提现审核中、提现成功三类。 | `DEC-MCH-002` |

## 2. MUST NOT

| 禁止项 | 决策ID |
|---|---|
| 不得把 `bizDomain` 写成 `FUND_PAYOUT_ORCHESTRATION`；资金出款业务域为 `FUND_PAYOUT`。 | `DEC-BIZ-004` |
| 不得在当前阶段设计 `currency / cur / currency_code` 字段。 | `DEC-CUR-001` |
| 不得使用 `payerIdentityRole / payer_identity_role`。 | `DEC-ROLE-001` |
| 支付通道网关不得重算提现 `amount`，不得推进出款状态，不得写出款结果事件。 | `DEC-AMT-006`、`DEC-GW-002` |
| 账户账务平台不得参与手续费、到账金额、内扣、外扣、通道提现金额计算。 | `DEC-AMT-004` |
| 本期不得新增 `account_statement_item` 表。 | `DEC-ACC-005` |
| 不得为 `prepareWithdrawCapability` 新增 `ChannelAccountController`、`/withdraw-capability/prepare` 或等价 HTTP Controller；当前单体阶段只允许单体内 Facade / ApplicationService 方法调用。 | `DEC-USER-20260601-WITHDRAW-CAPABILITY-INPROC-001` |
