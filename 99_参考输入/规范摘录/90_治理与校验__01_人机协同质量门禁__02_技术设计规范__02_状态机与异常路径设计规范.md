# 状态机与异常路径设计规范

## 1. 状态机不能只画成功路径

任何核心链路的状态机必须覆盖：

- 正常提交；
- 处理中；
- 成功；
- 失败；
- 未知；
- 超时；
- 重试中；
- 人工处理中；
- 可回正路径；
- 不可覆盖路径；
- 终态防覆盖。

## 2. 状态流转矩阵

状态机必须用矩阵表达，不仅用流程图。

| 当前状态 | 输入事件 | 是否允许 | 下一个状态 | 副作用 | 幂等规则 |
|---|---|---:|---|---|---|
| INIT | SUBMIT | 是 | PROCESSING | 创建下游请求 | 同幂等号返回已有记录 |
| PROCESSING | SUCCESS | 是 | SUCCESS | 触发成功副作用 | 重复 SUCCESS 不重复副作用 |
| PROCESSING | FAILED | 是 | FAILED | 触发失败副作用 | 重复 FAILED 不重复副作用 |
| PROCESSING | UNKNOWN | 是 | MANUAL_PROCESSING 或 RETRYING | 等待补偿 | 后续终局结果可回正 |
| SUCCESS | FAILED | 否 | SUCCESS | 无 | 记录迟到结果但不覆盖 |
| FAILED | SUCCESS | 视业务决定 | 通常不允许 | 无或人工处理 | 防乱序覆盖 |
| MANUAL_PROCESSING | SUCCESS | 视人工态来源 | 可能允许 | 继续收敛 | 仅允许 UNKNOWN 类人工态回正 |

## 3. 人工态必须说明来源

`MANUAL_PROCESSING`、`MANUAL_REQUIRED`、`EXCEPTION` 等状态不能作为垃圾桶。

必须说明：

| 来源 | 后续是否允许自动回正 |
|---|---|
| 外部结果未知 | 可允许后续明确结果回正 |
| 外部结果成功但本地副作用失败 | 不得再用外部结果覆盖，需修副作用 |
| 外部结果失败但本地回滚失败 | 不得再用外部结果覆盖，需修回滚 |
| 主关联缺失 | 不得自动猜测，进入人工 |
| 参数漂移 / 幂等冲突 | 不得自动覆盖，进入人工 |
| 数据模型不一致 | 不得继续推进，先修模型 |

## 4. 终态防覆盖

终态必须明确：

- 哪些事件允许重复接收；
- 哪些迟到事件只记录不处理；
- 哪些事件需要报警；
- 哪些事件可触发人工复核。

禁止没有判断地用“最新通知”覆盖当前终态。
