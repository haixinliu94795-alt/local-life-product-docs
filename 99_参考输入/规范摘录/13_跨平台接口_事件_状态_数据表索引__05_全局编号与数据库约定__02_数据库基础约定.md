# 06_数据库基础约定

本文定义平台化新表的数据库基础约定。各平台局部 DDL 可引用本文，不得定义冲突的主键、业务编号和跨表关联规则。

## 1. 主键与业务编号

平台化新表统一采用“双键”设计：

| 类型 | 规则 | 用途 |
|---|---|---|
| 物理主键 `id` | `BIGINT UNSIGNED NOT NULL AUTO_INCREMENT` | 仅作为数据库内部物理主键、聚簇索引和 ORM 内部标识。 |
| 业务编号 `*_no` | `VARCHAR(64)`，由所属平台应用层编号组件生成，通常为“前缀 + IdWorker 雪花ID” | 用于对外接口、跨平台协作、日志追踪、幂等定位、跨表业务关联。 |

约束：

1. `id` 不对外暴露，不进入接口契约，不作为跨表业务关联字段。
2. 不得使用数据库自增 `id` 作为业务编号。
3. 资金链路所有核心业务编号必须建立唯一索引。
5. 除非单独决策，否则平台化新表主键都必须按本文规则定义。

## 2. DDL 主键模板

```sql
id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
...
PRIMARY KEY (id)
```

## 3. 业务编号模板

```sql
xxx_no VARCHAR(64) NOT NULL COMMENT '业务编号，由所属平台应用层编号组件生成',
UNIQUE KEY uk_xxx_no (xxx_no)
```

业务编号生成规则以《05_全局编号规范》为准。本文只约束数据库主键与业务编号的职责边界。

## 4. 数据模型与建表 SQL 分层

数据模型文档表达领域事实、核心状态、关键关系和业务快照，不逐字段复制建表 SQL。

完整物理字段、索引、默认值、通用审计字段、逻辑删除字段以建表 SQL 为准。

如果字段影响以下内容，必须在数据模型中表达：

- 业务状态流转；
- 接口契约；
- 页面展示；
- 幂等与一致性；
- 人工处理；
- 跨平台关联；
- 历史展示锚点。

如果字段仅用于物理存储、索引、通用审计、逻辑删除或普通备注，可只在 DDL 中表达。
## 5. MySQL 字符集与排序规则统一约定

平台化新表统一采用：

```sql
ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci
```

约束：

1. 所有平台化新表必须显式声明 `DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci`，不得只写 `DEFAULT CHARSET=utf8mb4` 后依赖数据库默认排序规则。
2. 不得在 active DDL 中使用 `utf8mb4_unicode_ci`、`utf8mb4_0900_ai_ci` 或其他与 `utf8mb4_general_ci` 不一致的排序规则。
3. 跨表 Join 高频字段必须继承表级 `utf8mb4_general_ci`，包括 `identity_id`、`payer_identity_id`、`payee_identity_id`、`owner_identity_id`、`biz_no`、`*_no`、`channel_member_id`、`channel_order_no` 等。
4. 除非有单独 Decision Registry 裁决，不得在单列上声明与表级排序规则不同的 `COLLATE`。
5. 生产库如已存在不同排序规则，应通过受控 SQL 统一表或字段排序规则，禁止在业务 SQL 中长期通过 `COLLATE` 临时规避。

典型风险：

```sql
payment_channel_account.identity_id = fund_payout_process.payer_identity_id
```

如果两侧字段分别使用 `utf8mb4_unicode_ci` 与 `utf8mb4_general_ci`，MySQL 在 Join 时会报 `Illegal mix of collations`。因此 08 通道账户管理、03 资金出款编排、02 账户账务、05/06/07 支付平台相关 DDL 必须统一为 `utf8mb4_general_ci`。

已建表环境的整改 SQL 必须先评估锁表、索引重建和数据量风险，可采用表级转换或单列修正，例如：

```sql
ALTER TABLE payment_channel_account
  CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

ALTER TABLE fund_payout_process
  CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

如果只修正单列，必须确保 Join 两侧字段类型、长度、字符集、排序规则一致。
