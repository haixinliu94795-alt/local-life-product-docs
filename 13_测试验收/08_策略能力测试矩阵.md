# 策略能力测试矩阵

## 1. 本章结论

V009 增强了清分结算策略模型，因此 P0 必须增加策略默认值、策略绑定、策略匹配、策略快照和结算单金额口径测试。

## 2. 测试矩阵

| 用例编号 | 场景 | 输入 | 预期 |
|---|---|---|---|
| TC-POLICY-001 | 创建 P0 策略 | `INTERNAL_ACCOUNT + OPERATOR_MANUAL` | 成功 |
| TC-POLICY-002 | P0 策略误传自动结算 | `autoSettlementEnabled=true` | 拒绝或标记非 P0，不进入 P0 主链路 |
| TC-POLICY-003 | 创建 GLOBAL 绑定 | `bindingSubjectType=GLOBAL` | 绑定成功 |
| TC-POLICY-004 | 创建 MERCHANT 绑定 | 商户绑定特殊策略 | 策略匹配优先于 GLOBAL |
| TC-POLICY-005 | 策略匹配优先级 | GLOBAL + BUSINESS_SCENE + MERCHANT | 命中 MERCHANT |
| TC-POLICY-006 | 清分固化策略版本 | 履约事件命中策略 | `ClearingResult.policy_version_no` 有值 |
| TC-POLICY-007 | 头寸固化模式和发起方式 | 生成头寸 | `settlement_mode=INTERNAL_ACCOUNT`，`initiation_mode=OPERATOR_MANUAL` |
| TC-POLICY-008 | 默认无留存 | 生成结算单 | `reserve_amount_cent=0`，`posting_amount_cent=total_amount_cent` |
| TC-POLICY-009 | 最小金额默认不限制 | 策略 min=0 | 不拦截后台手动结算 |
| TC-POLICY-010 | request_hash 冲突 | 同一幂等键不同参数 | 返回幂等冲突错误 |

## 3. 测试边界

P0 不需要测试自动结算任务、商户自助结算、出款、留存归还和发票流程。
