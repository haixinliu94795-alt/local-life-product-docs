# 批量边界与 P0 约束测试

## 1. 本章结论

本测试不做性能压测，只验证 P0 的正确性边界：单批数量限制、全成功/全失败、任务单轮数量、导出非 P0。

## 2. 测试矩阵

| 用例编号 | 场景 | 输入 | 预期 |
|---|---|---|---|
| TC-BATCH-001 | 单条结算 | 1 个 `positionNo` | 成功生成结算单。 |
| TC-BATCH-002 | 正常批量结算 | 500 个 `positionNo` | 成功生成一个结算单，500 条明细。 |
| TC-BATCH-003 | 超过批量上限 | 501 个 `positionNo` | 返回 `CCS_BATCH_LIMIT_EXCEEDED`，不生成结算单。 |
| TC-BATCH-004 | 混入不可结算头寸 | 499 个 ELIGIBLE + 1 个 LOCKED | 整批失败，无状态变更。 |
| TC-BATCH-005 | 混入不同商户头寸 | 同一请求包含不同 merchantNo | 整批失败。 |
| TC-BATCH-006 | 账期成熟任务单轮限制 | 300 条到期 IN_TRANSIT | 单轮处理 200 条，其余下轮处理。 |
| TC-BATCH-007 | 导出接口 | 调用 `/eligible/export` | P0 无该接口或返回非 P0 能力。 |

## 3. 禁止将本章扩展为性能压测

本章不验证吞吐、TPS、百万数据、索引性能和容量规划。相关事项后续单独规划。
