# V010 开发准入结论

## 1. 结论

V010 达到 P0 开发准入要求。准入范围仅限 **本地生活/电商正向清结算闭环**，即来源事件、清分、结算头寸、账期成熟、结算确认、账务入账、失败补偿和基础查询。

## 2. 准入条件

| 条件 | 状态 |
|---|---|
| 领域模型清晰 | 已满足 |
| DDL 可执行 | 已满足 |
| 状态机闭环 | 已满足，新增账期成熟任务和 UNKNOWN SLA |
| 幂等规则闭合 | 已满足，新增 request_hash 标准 |
| 账务失败补偿 | 已满足 |
| 操作日志事务策略 | 已满足 |
| 性能容量规划 | 非 P0，不作为准入条件 |
| 大数据导出 | 非 P0，不作为准入条件 |

## 3. 准入限制

开发阶段必须遵守：

1. 不得新增冻结/止付状态。
2. 不得新增退款主链路。
3. 不得把商户端 BFF 展示口径写入底层状态机。
4. 不得把旧 `finance_detail` 当作领域模型。
5. 不得绕过 `SettlementAccountingPort` 直接依赖具体账务实现。
6. 不得绕过 `request_hash` 做幂等判断。
7. 不得实现 `/eligible/export` 作为 P0 核心接口。
