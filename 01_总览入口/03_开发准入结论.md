# 开发准入结论

## 结论

V008 可以作为 P0 普通商品正向清结算的开发准入方案。进入代码前需要研发按本包任务卡拆分执行，不允许直接跳过 DDD、状态机、DDL、接口契约做 CRUD。

## 准入范围

| 模块 | 是否准入 |
|---|---|
| 清分结算策略配置 | 是 |
| 来源事件接入 | 是 |
| 清分结果与清分明细 | 是 |
| 结算头寸 | 是 |
| 结算单与批次 | 是 |
| 账务入账编排 | 是 |
| 商户端 BFF 展示适配 | 否，本版只提供标准查询能力 |
| 退款/冲正 | 否 |
| 冻结/止付 | 否 |
| 自动出款 | 否 |
| 上线部署 | 否 |
