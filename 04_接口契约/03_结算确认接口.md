# 结算确认接口

## 1. 统一接口

```text
POST /admin/ccs/settlement-bills/confirm
```

单条结算和批量结算都调用该接口：

```text
单条：positionNos.size() == 1
批量：positionNos.size() > 1
```

## 2. ConfirmMerchantSettlementCommand

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `callerSystem` | String | 是 | 调用系统。 |
| `bizDomain` | String | 是 | 业务域。 |
| `bizNo` | String | 是 | 本次结算确认业务请求号。 |
| `merchantNo` | String | 是 | 结算商户。 |
| `businessScene` | String | 是 | 业务场景。 |
| `positionNos` | List<String> | 是 | 待结算头寸编号。 |
| `operatorId` | String | 是 | 操作人。 |
| `operatorName` | String | 否 | 操作人名称。 |
| `voucherUrl` | String | 否 | 结算凭证。 |
| `remark` | String | 否 | 备注。 |
| `idempotentKey` | String | 是 | 幂等键。 |
| `requestHash` | String | 是 | 请求指纹。 |

## 3. 批量校验

批量结算必须满足：

1. 全部头寸状态为 `ELIGIBLE`。
2. 全部头寸属于同一 `merchantNo`。
3. 全部头寸属于同一 `businessScene`。
4. 全部头寸未被其他结算单锁定。
5. 批量策略为全成功/全失败。
