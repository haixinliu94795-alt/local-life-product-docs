# 事件接入契约

## 1. SettlementSourceEventCommand

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `callerSystem` | String | 是 | 调用系统。 |
| `bizDomain` | String | 是 | 业务域，例如 `LOCAL_LIFE`。 |
| `bizNo` | String | 是 | 来源业务请求号。 |
| `eventType` | String | 是 | P0 固定 `FULFILLMENT_CONFIRMED`。 |
| `businessScene` | String | 是 | 业务场景，例如 `LOCAL_LIFE_GOODS`。 |
| `sourceOrderNo` | String | 是 | 来源订单号。 |
| `sourceFulfillmentNo` | String | 是 | 来源履约/核销号。 |
| `merchantNo` | String | 是 | 商户编号。 |
| `fulfillmentTime` | DateTime | 是 | 履约/核销完成时间。 |
| `orderAmountCent` | Long | 是 | 订单金额，单位分。 |
| `paidAmountCent` | Long | 是 | 用户实付金额，单位分。 |
| `merchantReceivableCent` | Long | 是 | 商户应结金额，单位分。 |
| `platformReceivableCent` | Long | 是 | 平台应收金额，单位分。 |
| `discountCostCent` | Long | 否 | 优惠成本，单位分。 |
| `idempotentKey` | String | 是 | 来源事件幂等键。 |
| `requestHash` | String | 是 | 请求核心参数指纹。 |

## 2. 幂等规则

- 同一 `idempotentKey + requestHash` 重复提交，返回同一 `event_no`。
- 同一 `idempotentKey` 但 `requestHash` 不同，返回 `CCS_IDEMPOTENT_CONFLICT`。
- 同一 `sourceOrderNo + sourceFulfillmentNo + eventType` 只能处理一次。

## 3. 不支持事件

P0 不接入退款、撤销、提现、冻结、止付事件。
