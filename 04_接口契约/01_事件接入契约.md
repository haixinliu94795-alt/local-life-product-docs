# 事件接入契约

## 1. SourceEventAcceptCommand

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `callerSystem` | String | 是 | 技术调用方系统。 |
| `bizDomain` | String | 是 | 业务归属域。 |
| `bizNo` | String | 是 | 业务单号。 |
| `businessScene` | String | 是 | 业务场景。 |
| `eventType` | String | 是 | `WRITE_OFF_COMPLETED/REFUND_SUCCESS/WRITE_OFF_CANCELED`。 |
| `sourceEventNo` | String | 是 | 来源事件唯一号。 |
| `sourceOrderNo` | String | 否 | 来源订单号。 |
| `sourceWriteOffNo` | String | 否 | 来源核销号。 |
| `sourceRefundNo` | String | 否 | 来源退款号。 |
| `eventOccurTime` | DateTime | 是 | 事件发生时间。 |
| `idempotentKey` | String | 是 | 幂等键。 |
| `requestHash` | String | 是 | 请求指纹。 |
| `payload` | Object | 是 | 原始事件载荷。 |

## 2. 幂等处理

- 同 `idempotentKey` + 同 `requestHash`：返回原 `eventNo`。
- 同 `idempotentKey` + 不同 `requestHash`：返回 `CCS_IDEMPOTENT_CONFLICT`。
