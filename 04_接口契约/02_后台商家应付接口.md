# 后台商家应付接口

## 1. 商家应付分页

只允许查询 `ccs_pending_settlement_item.lifecycle_status = ELIGIBLE`。

## 2. 统一确认结算

接口：`POST /admin/ccs/merchant-payable/settle`

核心方法：

```java
SettlementBillNo confirmMerchantSettlement(ConfirmMerchantSettlementCommand command);
```

单条与批量区别仅在 `pendingNos` 数量：

- 单条：`pendingNos.size() == 1`
- 批量：`pendingNos.size() > 1`

P0 批量策略：全成功/全失败。

## 3. 批量校验

| 校验 | 说明 |
|---|---|
| 非空 | `pendingNos` 不能为空。 |
| 同一商户 | 所有待结算项必须属于同一商户。 |
| 同一结算主体 | 所有待结算项必须属于同一结算主体。 |
| 同一业务场景 | P0 只允许普通商品同场景。 |
| 状态正确 | 必须全为 `ELIGIBLE`。 |
| 金额正确 | 汇总金额等于明细金额之和。 |
| 幂等一致 | 同幂等键不同 requestHash 拒绝。 |
