# 账务平台对接契约

## 1. 设计原则

清结算平台不绑定当前任何已有 facade 方法名。P0 需要账户账务平台提供标准端口：

```java
SettlementAccountingPostResult postSettlement(SettlementAccountingPostCommand command);
SettlementAccountingPostResult querySettlementPosting(String accountingRequestNo);
```

如果账户账务平台已有旧接口，可由账户账务平台或适配器实现该标准端口，不反向修改清结算平台领域模型。

## 2. SettlementAccountingPostCommand

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `callerSystem` | String | 是 | `CLEARING_SETTLEMENT_PLATFORM`。 |
| `bizDomain` | String | 是 | 业务域。 |
| `bizNo` | String | 是 | 结算单号 `bill_no`。 |
| `accountingScene` | String | 是 | `MERCHANT_SETTLEMENT_POSTING`。 |
| `settlementTargetType` | String | 是 | P0 为 `MERCHANT`。 |
| `settlementTargetNo` | String | 是 | 商户编号。 |
| `postingAmountCent` | Long | 是 | 入账金额，单位分。 |
| `idempotentKey` | String | 是 | 账务幂等键。 |
| `requestHash` | String | 是 | 账务请求指纹。 |
| `billNo` | String | 是 | 清结算结算单号。 |
| `batchNo` | String | 是 | 清结算批次号。 |

## 3. SettlementAccountingPostResult

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `postingStatus` | String | 是 | `SUCCESS/FAILED/UNKNOWN`。 |
| `accountingRequestNo` | String | 成功或已受理时必填 | 账户账务请求号。 |
| `fundAccountFlowNo` | String | 成功时必填 | 账户流水号或等价可追踪流水引用。 |
| `failCode` | String | 失败时必填 | 失败码。 |
| `failMessage` | String | 失败时必填 | 失败原因。 |

## 4. 回写要求

清结算平台必须将以下字段回写到 `ccs_accounting_posting_order` 和 `ccs_settlement_bill`：

```text
accounting_request_no
fund_account_flow_no
posting_status
posting_success_time
fail_reason
```

## 5. UNKNOWN 处理

当调用超时、网络异常或账户账务平台返回结果不明时，清结算平台进入 `UNKNOWN`，只能通过 `querySettlementPosting` 查询结果，不允许直接再次入账。
