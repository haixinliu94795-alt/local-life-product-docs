# 后台运营接口

| 接口 | 说明 | 一期 |
|---|---|---:|
| `GET /admin/clear-settlement/payable-items` | 商家应付/待结算列表。 | 是 |
| `POST /admin/clear-settlement/batches/preview` | 结算预览和校验。 | 是 |
| `POST /admin/clear-settlement/bills/confirm` | 上传凭证并确认结算。 | 是 |
| `GET /admin/clear-settlement/bills` | 结算记录列表。 | 是 |
| `GET /admin/clear-settlement/bills/{billNo}` | 结算详情。 | 是 |
| `POST /admin/clear-settlement/bills/{billNo}/retry-accounting` | 入账失败重试。 | 是 |
| `POST /admin/clear-settlement/pending-items/freeze` | 冻结待结算项。 | P1 |
| `POST /admin/clear-settlement/pending-items/unfreeze` | 解冻待结算项。 | P1 |
| `POST /admin/clear-settlement/rules/publish` | 发布结算规则。 | 一期简化 |

## 关键校验

- 批量结算必须至少选择一条数据；
- 批量结算必须同一商户；
- 批量结算必须同一业务场景/结算场景；
- 已结算、已锁定、冻结项不能重复结算；
- 结算金额由服务端重算；
- 凭证最多 5 张，格式按当前原型约束；
- 所有写操作记录操作日志。
