# 字段设计说明

## 1. 本章结论

V009 的字段设计坚持资金系统字段规范：金额统一为 `BIGINT` 分，人民币固定口径，不设计 `currency`，不设计 `tenant_id`，所有写入链路必须有业务键、幂等键和请求指纹。

本版新增字段集中在清分结算策略和结算单金额口径上，目的是吸收行业通用能力，但不扩大 P0 实现范围。

## 2. 策略版本新增字段

| 字段 | 表 | 说明 | P0 默认 |
|---|---|---|---|
| `settlement_mode` | `ccs_settlement_policy_version` | 结算模式：内部账户、直接银行、外部出款 | `INTERNAL_ACCOUNT` |
| `initiation_mode` | `ccs_settlement_policy_version` | 发起方式：后台手动、自动、商户自助 | `OPERATOR_MANUAL` |
| `auto_settlement_enabled` | `ccs_settlement_policy_version` | 是否自动结算 | `0` |
| `min_settlement_amount_cent` | `ccs_settlement_policy_version` | 最小结算金额 | `0` |
| `max_settlement_amount_cent` | `ccs_settlement_policy_version` | 最大结算金额 | `0` |
| `reserve_policy_type` | `ccs_settlement_policy_version` | 留存策略 | `NONE` |
| `reserve_amount_cent` | `ccs_settlement_policy_version` | 固定留存金额 | `0` |
| `reserve_ratio_bp` | `ccs_settlement_policy_version` | 留存比例 | `0` |

## 3. 策略绑定表

`ccs_settlement_policy_binding` 表示业务对象引用策略，不复制策略内容。

| 字段 | 说明 |
|---|---|
| `binding_subject_type` | 绑定对象类型：GLOBAL/BUSINESS_SCENE/CATEGORY/MERCHANT/CHANNEL/CAMPAIGN |
| `binding_subject_no` | 绑定对象编号，GLOBAL 使用 `*` |
| `policy_no` | 引用策略编号 |
| `binding_priority` | 优先级，越小越优先 |
| `effective_start_time` / `effective_end_time` | 生效区间 |
| `idempotent_key` / `request_hash` | 写入幂等和冲突判断 |

## 4. 结算头寸新增固化字段

`ccs_settlement_position` 增加：

```text
policy_no
policy_version_no
settlement_mode
initiation_mode
```

它们用于固化头寸生成时命中的策略口径。后续策略修改不影响已生成头寸。

## 5. 结算单金额口径

`ccs_settlement_bill` 固化：

| 字段 | 说明 |
|---|---|
| `total_amount_cent` | 结算明细汇总金额 |
| `reserve_amount_cent` | 留存金额，P0 默认 0 |
| `posting_amount_cent` | 本次应入账金额，等于 `total_amount_cent - reserve_amount_cent` |
| `min_settlement_amount_cent` | 生成结算单时适用的最小金额策略 |
| `max_settlement_amount_cent` | 生成结算单时适用的最大金额策略 |

P0 中 `reserve_amount_cent=0`，因此 `posting_amount_cent=total_amount_cent`。

## 6. 字段设计禁令

P0 不得新增：

```text
currency / cur / currency_code
tenant_id
refund_no / refund_amount
freeze / hold / stop_pay
payout_order_no
invoice_no
```

这些字段如后续需要，必须通过单独 ADR 和新版本方案引入。
