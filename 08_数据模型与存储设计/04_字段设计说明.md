# 字段设计说明

## 1. 本章结论

V010 字段设计遵循资金系统 P0 正确性优先原则：金额统一使用分单位整数；写请求必须有幂等键和 request_hash；状态字段必须能与状态机矩阵对应；UNKNOWN 补偿和账期成熟任务需要最小可观测字段。

## 2. 全局字段规范

| 字段类型 | 规范 |
|---|---|
| 主键 | `id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT` |
| 业务编号 | `xxx_no VARCHAR(64)`，外部订单号可 `VARCHAR(128)` |
| 金额 | `BIGINT`，单位分，人民币 |
| 比例 | `ratio_bp INT`，10000 = 100% |
| 状态 | `VARCHAR(32)`，枚举值与状态机文件一致 |
| 时间 | `DATETIME(3)` |
| 幂等 | `idempotent_key VARCHAR(128)` |
| 请求指纹 | `request_hash VARCHAR(128)`，SHA-256 小写十六进制 |
| 逻辑删除 | `deleted TINYINT NOT NULL DEFAULT 0` |
| 乐观锁 | `version BIGINT NOT NULL DEFAULT 0` |

P0 不设计 `currency` 字段，不设计 `tenant_id` 字段。

## 3. 关键字段说明

### 3.1 `ccs_settlement_position`

| 字段 | 说明 | 设计目的 |
|---|---|---|
| `position_no` | 结算头寸编号 | 结算前后的核心追踪编号。 |
| `clearing_item_no` | 清分明细编号 | 保证一个清分明细只生成一个头寸。 |
| `eligible_time` | 可结算时间 | 账期成熟任务扫描依据。 |
| `last_maturity_attempt_time` | 最近成熟推进尝试时间 | 观察成熟任务是否运行。 |
| `maturity_attempt_count` | 成熟推进尝试次数 | 支持失败排查，不做容量指标。 |
| `lifecycle_status` | 头寸生命周期状态 | 对应 `SettlementPosition` 状态机。 |
| `locked_batch_no / locked_bill_no` | 锁定批次/结算单 | 防止重复结算。 |

### 3.2 `ccs_accounting_posting_order`

| 字段 | 说明 | 设计目的 |
|---|---|---|
| `accounting_idempotent_key` | 账务幂等键 | 账务重试和 UNKNOWN 查询核心依据。 |
| `unknown_first_time` | 首次进入 UNKNOWN 时间 | 判断 UNKNOWN 持续时长。 |
| `last_query_time` | 最近账务查询时间 | 观察补偿任务。 |
| `next_query_time` | 下次账务查询时间 | UNKNOWN 补偿任务扫描依据。 |
| `query_attempt_count` | 查询次数 | 最大自动查询次数判断。 |
| `manual_intervention_required` | 是否需要人工介入 | UNKNOWN 超 SLA 后诊断入口。 |

### 3.3 `ccs_operation_log`

操作日志为审计辅助数据。日志写入不参与资金主事务成败判定。字段只用于记录聚合状态变更和操作上下文。

## 4. 非 P0 字段不进入 DDL

以下字段不进入 V010 P0：

```text
currency / tenant_id
freeze_status / hold_status
refund_no / refund_amount
payout_no / withdraw_no
export_task_no
sharding_key
```
