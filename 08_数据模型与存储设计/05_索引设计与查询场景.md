# 索引设计与查询场景

## 本章结论

索引必须对应具体查询场景，不能为了“可能有用”随意添加。

| 查询场景 | 表 | 索引 |
|---|---|---|
| 事件幂等 | `ccs_source_event` | `uk_event_idempotent`、`uk_event_biz` |
| 清分结果反查 | `ccs_clearing_result` | `uk_clearing_event`、`idx_clearing_order` |
| 账期成熟扫描 | `ccs_settlement_position` | `idx_position_eligible(lifecycle_status, eligible_time, id)` |
| 商家应付查询 | `ccs_settlement_position` | `idx_position_merchant` |
| 结算单查询 | `ccs_settlement_bill` | 按 `merchant_no/business_scene/bill_status/create_time` |
| 入账单幂等 | `ccs_accounting_posting_order` | `idempotent_key`、业务键组合 |
