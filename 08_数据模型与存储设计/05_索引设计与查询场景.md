# 索引设计与查询场景

## 1. 本章结论

V009 的索引设计围绕资金链路的四类核心访问场景：事件幂等、策略匹配、可结算头寸查询、结算单与账务追踪。

## 2. 策略匹配索引

| 表 | 索引 | 典型查询 |
|---|---|---|
| `ccs_settlement_policy` | `idx_policy_scope_status` | 按作用域、业务场景和生效时间查策略 |
| `ccs_settlement_policy_binding` | `idx_binding_subject` | 按对象类型、对象编号、业务场景匹配绑定策略 |
| `ccs_settlement_policy_binding` | `idx_binding_policy` | 按策略反查绑定对象 |
| `ccs_policy_rule` | `idx_rule_policy` | 按策略版本查规则集 |

策略匹配顺序由应用层控制，建议为：

```text
MERCHANT > CAMPAIGN > CATEGORY > BUSINESS_SCENE > GLOBAL
```

## 3. 头寸查询索引

后台“商家应付”查询只查可结算头寸：

```sql
SELECT *
FROM ccs_settlement_position
WHERE lifecycle_status = 'ELIGIBLE'
  AND business_scene = ?
  AND merchant_no = ?
ORDER BY eligible_time, id
LIMIT ?, ?;
```

对应索引：

```text
idx_position_eligible(lifecycle_status, eligible_time, id)
idx_position_merchant(merchant_no, business_scene, lifecycle_status)
```

## 4. 结算追踪索引

资金追踪主链路：

```text
event_no -> clearing_no -> clearing_item_no -> position_no -> bill_no -> posting_no -> accounting_request_no
```

需要的索引：

| 表 | 索引 |
|---|---|
| `ccs_source_event` | `uk_event_biz`, `uk_event_idempotent` |
| `ccs_clearing_result` | `uk_clearing_event` |
| `ccs_clearing_result_item` | `idx_item_position` |
| `ccs_settlement_bill_item` | `uk_bill_item_position` |
| `ccs_accounting_posting_order` | `uk_posting_bill`, `uk_accounting_idempotent` |

## 5. 金额阈值字段查询

最小/最大金额和留存字段主要用于策略执行和结算单回溯，不作为高频查询条件，不单独加索引。
