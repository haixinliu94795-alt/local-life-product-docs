# 索引设计与查询场景

## 1. 本章结论

V010 索引只覆盖 P0 正确性和开发期常用查询，不做容量规划和复杂性能优化。索引用于保证唯一性、防重、状态推进、后台分页查询和 UNKNOWN 补偿。

## 2. 唯一索引

| 表 | 唯一索引 | 目的 |
|---|---|---|
| `ccs_source_event` | `uk_event_no` | 事件编号唯一。 |
| `ccs_source_event` | `uk_event_idempotent` | 接入幂等。 |
| `ccs_source_event` | `uk_event_biz` | 来源业务唯一约束。 |
| `ccs_clearing_result` | `uk_clearing_event` | 一个事件只能生成一份清分结果。 |
| `ccs_settlement_position` | `uk_position_clearing_item` | 一个清分明细只能生成一个头寸。 |
| `ccs_settlement_bill` | `uk_bill_idempotent` | 结算确认幂等。 |
| `ccs_settlement_bill_item` | `uk_bill_item_position` | 一个头寸只能进入一个结算明细。 |
| `ccs_accounting_posting_order` | `uk_posting_bill` | 一个结算单只能有一个入账单。 |
| `ccs_accounting_posting_order` | `uk_accounting_idempotent` | 账务入账幂等。 |

## 3. 关键查询索引

### 3.1 账期成熟任务

```sql
KEY idx_position_eligible (lifecycle_status, eligible_time, id)
```

用于：

```sql
where lifecycle_status = 'IN_TRANSIT'
  and eligible_time <= now()
order by id asc
limit 200
```

### 3.2 后台可结算头寸分页

```sql
KEY idx_position_merchant (merchant_no, business_scene, lifecycle_status)
```

用于后台查询 `ELIGIBLE` 头寸。

### 3.3 UNKNOWN 补偿任务

```sql
KEY idx_posting_unknown_query (posting_status, next_query_time, manual_intervention_required)
```

用于：

```sql
where posting_status = 'UNKNOWN'
  and next_query_time <= now()
  and manual_intervention_required = 0
```

## 4. 不做事项

本章不设计：

```text
分库分表索引
覆盖索引性能优化
冷热归档索引
百万级导出索引
大商户专属索引
```
