# 数据保留与非 P0 容量规划说明

## 1. 本章结论

V010 不进行容量规划。本文只说明哪些能力不是 P0，避免研发在 P0 中误做性能工程。

## 2. P0 数据保留

P0 所有清结算主表均在线保留，不做自动归档。

```text
ccs_source_event
ccs_clearing_result
ccs_clearing_result_item
ccs_settlement_position
ccs_settlement_batch
ccs_settlement_bill
ccs_settlement_bill_item
ccs_accounting_posting_order
ccs_operation_log
```

## 3. 非 P0 规划

以下能力后续单独设计：

| 能力 | 状态 |
|---|---|
| 历史归档 | 非 P0 |
| 分表分库 | 非 P0 |
| 亿级明细查询 | 非 P0 |
| 离线报表 | 非 P0 |
| 异步导出 | 非 P0 |
| 容量评估 | 非 P0 |
| 压测指标 | 非 P0 |

## 4. P0 最小防护

| 防护 | 数值 |
|---|---:|
| 成熟任务单轮处理数 | 200 |
| 结算确认单次头寸数 | 500 |
| 查询最大 pageSize | 100 |

这些防护用于避免误用，不代表容量指标。
