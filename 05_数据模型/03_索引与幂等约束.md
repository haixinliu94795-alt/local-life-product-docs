# 索引与幂等约束

## 1. 幂等唯一约束

| 表 | 约束 |
|---|---|
| `ccs_source_event` | `idempotent_key`、`caller_system + biz_domain + biz_no`、`source_order_no + source_fulfillment_no + event_type` |
| `ccs_clearing_result` | `event_no`、`idempotent_key` |
| `ccs_settlement_position` | `clearing_item_no` |
| `ccs_settlement_bill` | `caller_system + biz_domain + biz_no`、`idempotent_key` |
| `ccs_settlement_bill_item` | `position_no` |
| `ccs_accounting_posting_order` | `bill_no`、`idempotent_key`、`accounting_idempotent_key` |

## 2. 请求指纹

写入类接口必须携带 `request_hash`。处理规则：

| 场景 | 处理 |
|---|---|
| 幂等键相同、请求指纹相同 | 返回原结果。 |
| 幂等键相同、请求指纹不同 | 拒绝，返回 `CCS_IDEMPOTENT_CONFLICT`。 |
| 幂等键不同、业务键重复 | 返回已存在业务结果或拒绝，按接口契约处理。 |

## 3. 查询索引

重点索引：

```text
ccs_settlement_position(lifecycle_status, eligible_time)
ccs_settlement_position(settlement_target_type, settlement_target_no, lifecycle_status)
ccs_settlement_bill(settlement_target_type, settlement_target_no, create_time)
ccs_accounting_posting_order(posting_status, create_time)
```
