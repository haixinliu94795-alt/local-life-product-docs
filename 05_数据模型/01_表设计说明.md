# 表设计说明

## 1. 核心变化

V005 将 V004 的 `ccs_pending_settlement_item` 调整为 `ccs_settlement_position`。原因：

- `pending` 容易和产品展示“待结算”混淆；
- `position` 更符合清结算领域的头寸概念；
- 结算头寸可以表达在途、可结算、锁定、入账中、入账失败、未知、已入账等完整生命周期。

## 2. 金额字段

所有金额字段：

```text
BIGINT，单位分，人民币
```

不设计币种字段。

## 3. 旧系统字段

DDL 不包含旧系统字段映射，不出现旧表名。旧系统接入标准事件由适配层处理。

## 4. 退款字段

P0 不设计退款字段。未来逆向清分能力通过独立事件和独立调整表扩展，不污染正向清结算主链路。
