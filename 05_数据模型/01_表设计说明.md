# 表设计说明

## 1. 主关联对象

清结算 P0 的主关联对象是 `SettlementBill`，主关联编号是 `bill_no`。

## 2. 关键业务编号

| 编号 | 表 | 说明 |
|---|---|---|
| `event_no` | `ccs_source_event` | 来源事件号。 |
| `clearing_no` | `ccs_clearing_result` | 清分结果号。 |
| `clearing_item_no` | `ccs_clearing_result_item` | 清分明细号。 |
| `pending_no` | `ccs_pending_settlement_item` | 待结算项号。 |
| `batch_no` | `ccs_settlement_batch` | 结算批次号。 |
| `bill_no` | `ccs_settlement_bill` | 结算单号。 |
| `posting_no` | `ccs_accounting_posting_order` | 账务入账单号。 |

## 3. request_hash 规则

`request_hash` 不参与唯一索引。它用于在同一 `idempotent_key` 下判断请求参数是否一致。

- 一致：返回原业务结果。
- 不一致：拒绝，返回 `CCS_IDEMPOTENT_CONFLICT`。
