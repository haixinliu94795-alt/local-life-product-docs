# 设计原则与非目标

## 1. 设计原则

| 原则 | 说明 |
|---|---|
| 领域优先 | 按业务接入、清分、清算、结算、账务、对账划分上下文，不按页面和旧表划分。 |
| 业务线隔离 | 本地生活、按摩、渠道只做 adapter，不改清结算核心。 |
| 单据驱动 | 清分单、清算批次、结算单、账务入账单形成可追溯链路。 |
| 状态可追踪 | 所有资金动作必须有状态机，不允许“点击即成功”。 |
| 幂等优先 | 写链路先幂等，再重试，避免重复结算和重复入账。 |
| 读写分离 | 写模型保证一致性，页面查询通过投影模型承接。 |
| 对账可用 | 每层保留对账键、金额快照和状态快照。 |
| 合规边界 | 清结算平台不直接沉淀资金，不替代账户账务和出款平台。 |

## 2. 一期非目标

- 不做自动出款；
- 不做完整税票系统；
- 不做完整对账差错工作台；
- 不做多级分润规则引擎；
- 不做历史数据全量迁移；
- 不直接替换账户账务平台；
- 不把 `local_settlement_record` 扩展成新平台核心。
