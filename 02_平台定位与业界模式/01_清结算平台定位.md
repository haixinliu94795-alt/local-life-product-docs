# 清结算平台定位

## 1. 一句话定位

清结算平台是交易后资金归属与结算事实平台，负责把业务交易、履约、核销、退款等事件转化为清分结果、清算结果、待结算项、结算单和账务入账指令，并通过对账差错机制保障资金链路一致。

## 2. 为什么不是单纯“结算平台”

当前需求表面上是“商家应付、结算、结算记录”，但成熟平台不能只做结算记录。否则本地生活、按摩、团餐、渠道、邀新每来一个业务都会复制一套结算逻辑。

本平台需要覆盖三层：

| 层 | 定义 | 输出 |
|---|---|---|
| 清分 | 单笔交易维度，把金额拆给商户、平台、渠道、技师、门店、服务商等参与方。 | `ClearingResult`、`ClearingResultItem` |
| 清算 | 周期维度，对清分结果按账期、商户、业务线、状态做成熟判断、汇总、冻结、锁定和轧差。 | `ClearingBatch`、`PendingSettlementItem`、应收/应付汇总 |
| 结算 | 单据维度，生成结算批次、结算单、结算明细，编排账务入账，并提供商户展示和对账。 | `SettlementBatch`、`SettlementBill`、`AccountingOrder` |

## 3. 平台边界

| 系统 | 负责 | 不负责 |
|---|---|---|
| 订单/业务平台 | 订单、履约、核销、退款、业务状态 | 不计算最终商户余额，不生成结算单 |
| 支付平台 | 收单、支付、退款、渠道账单 | 不决定商户应结金额 |
| 清结算平台 | 清分、清算、待结算、结算单、账务入账编排、对账差错 | 不保存账户余额最终事实，不直接银行卡出款 |
| 账户账务平台 | 资金主体、账户余额、账户流水、账务请求幂等 | 不决定哪些订单该结算 |
| 出款/提现平台 | 提现审核、扣款、银行卡出款、出款状态 | 不生成结算单 |
| 商户端/BFF | 展示余额、待结算、结算记录、提现记录 | 不承载清结算核心规则 |

## 4. 合规边界

清结算平台只做信息流、账务指令流和内部单据流，不自行沉淀客户备付金，不绕过支付机构或银行进行真实资金划拨。真实余额事实在账户账务平台，真实出款在出款平台或合规支付/银行通道。
