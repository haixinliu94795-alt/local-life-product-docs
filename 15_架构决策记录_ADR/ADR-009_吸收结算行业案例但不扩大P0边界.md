# ADR-009 吸收结算行业案例但不扩大 P0 边界

## 状态

Accepted

## 背景

补充参考的个人博主文章覆盖企业福利、推广平台、银行收单、积分电商、ETC 等案例。文章中的部分能力属于清结算平台通用能力，但很多能力属于传统行业的收款、付款、发票、合同、验收或税务流程。

## 备选方案

| 方案 | 说明 | 结果 |
|---|---|---|
| 全量吸收 | 把文章中的发票、收款、付款、税票、合同等能力都并入清结算平台 | 拒绝，边界过重 |
| 完全不吸收 | 认为文章非电商/本地生活，不参考 | 拒绝，会遗漏结算模式、周期、阈值、留存等通用能力 |
| 裁剪吸收 | 只吸收通用结算能力，非主链路能力留在外部平台或 P1/P2 | 采用 |

## 决策

V010 采用裁剪吸收：

- 吸收结算模式、结算周期、发起方式、最小/最大金额、留存金额、规则绑定对象、账单和对账意识。
- 不吸收传统行业发票、收款、付款、合同、验收、税务流程进入 P0。
- 所有新增能力落在 `ClearingSettlementPolicy` 策略上下文，不改变 P0 普通商品正向清结算闭环。

## 影响

- 策略版本表新增模式、发起、阈值、留存字段。
- 新增策略绑定表。
- P0 默认关闭自动结算、商户自助、留存和出款。
- 后续扩展无需推翻策略模型。
